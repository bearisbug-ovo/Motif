# 导入流程

> 涉及代码：`backend/routers/media.py`(import endpoints) | `frontend/src/components/ImportDialog.tsx, ClipboardImportDialog.tsx` | `frontend/src/stores/media.ts`

---

## 需求摘要

### 导入触发方式 [PRD §6.1]

- **粘贴导入：** 捕获全局剪贴板粘贴事件（`Ctrl+V`），导入单张图片，保存至 `AppData/imports/clipboard/`
- **文件选择（桌面端）：** 唤起系统文件选择器（tkinter），支持选择单张/多张媒体文件或文件夹。文件选择器默认显示"媒体文件"过滤器，包含图片和视频格式
- **文件上传（移动端）：** 移动端（触屏设备）使用 HTML `<input type="file" multiple accept="image/*,video/*">` 替代 tkinter 文件选择器，文件通过 HTTP multipart 上传到服务器，保存至 `AppData/imports/upload/`。移动端不支持子文件夹模式

### 导入逻辑 [PRD §6.2]

**选择单张或多张图片：**
1. 询问关联人物（链接到已有人物 / 新建人物 / 暂时无人物）
2. 询问是否新建图集或暂时无图集

**选择无子文件夹的文件夹：**
1. 将该文件夹视为一个图集导入
2. 询问是否更改图集名称（默认使用文件夹名）
3. 询问关联人物

**选择有子文件夹的文件夹（递归处理）：**
1. 展开导入确认弹窗，列出所有子文件夹（多层嵌套全部展平）
2. 每行显示：文件夹路径 → 图集名称（可编辑）→ 所属人物（可选）
3. 用户逐行确认或批量设置后提交

**文件夹名称智能拆分：**
- 当文件夹名包含下划线 `_`、横杠 `-` 或空格时，按**第一个**分隔符拆分
- 分隔符左侧作为默认**人物名**，右侧作为默认**图集名**
- 示例：`Alice_Wedding` → 人物"Alice"，图集"Wedding"；`Bob-旅行照片` → 人物"Bob"，图集"旅行照片"
- 若文件夹名不含上述分隔符，则人物名和图集名均默认为完整文件夹名
- 此规则适用于所有导入场景：无子文件夹的单文件夹、有子文件夹展平后的每一行、选择多张文件时的父文件夹名

**人物自动匹配与去重：**
- 拆分或推导出的人物名若与已有人物**名称匹配**（不区分大小写），自动选中该已有人物，而非新建
- 多个子文件夹拆分出相同人物名时（如 `Alice_生活照` 和 `Alice_写真集`），仅在第一次创建人物，后续自动复用，不会创建重复人物
- 用户仍可在导入弹窗中手动修改人物关联

### 导入进度与取消 [PRD §6.3]

- **进度显示：** 导入过程中显示进度条和计数（如"正在导入 45 / 230"）
- **可取消：** 导入过程中可随时取消（点击"取消导入"按钮），已导入的记录保留（不回滚）
- **导入中保护：** 导入进行中时，点击对话框外部或按 Escape 键不会关闭对话框，防止意外中断
- **大批量处理：** 超过 500 张图片时转为后台任务，顶部显示进度通知，用户可继续操作其他功能

### 人物上下文默认值 [PRD §6.4]

- 在**主页**触发导入：询问用户选择人物
- 在**人物主页**触发导入：默认关联当前人物，用户可强制更改

### 去重 [PRD §6.5]

- 以文件绝对路径作为唯一标识去重
- 重复扫描时跳过已存在记录，不重复导入
- **导入前预览：** 选择文件夹/文件后，导入界面立即显示媒体总数和已存在数量（异步扫描，不阻塞 UI）
  - 单文件夹/选择文件模式：来源区域下方显示"共 X 个媒体文件，其中 Y 个已存在将跳过"
  - 子文件夹模式：每个子文件夹行显示该文件夹的媒体数和已存在数；标题汇总全部文件夹的总数和已存在数
  - 全部已存在的文件夹自动排到列表末尾并降低不透明度，优先展示含新文件的文件夹
  - **子文件夹自动取消勾选**：扫描完成后，若子文件夹内所有文件均已存在（`existing >= total`），该子文件夹的复选框自动取消勾选
- **逐个配置模式已导入标记**：逐个配置（per-media）和子文件夹逐个视图中，后端 `POST /api/media/list-files` 返回每个文件的 `existing` 状态；已导入的文件在文件名旁显示"已导入"琥珀色标签，复选框置灰禁用且默认不勾选；全选操作自动跳过已导入文件
- **空图集清理**：导入流程中新创建的图集 ID 被追踪，导入完成后（包括错误/取消情况）自动检查并删除未导入任何媒体的空图集，避免产生垃圾数据
- **回收站重导入**：导入已在回收站中的文件时，自动恢复软删除记录（而非创建重复记录），更新 person_id 和 album_id 为新目标

### 文件丢失处理 [PRD §6.6, P1]

- 系统检测到文件路径不可访问时，卡片显示丢失标记（红色虚线边框 + 警告图标）
- 大图浏览（LightBox）中图片加载失败时，显示错误占位 UI（AlertTriangle 图标 + "文件不存在或无法加载"提示文字）
- 支持多入口重新定位：
  - **MediaCard 右键菜单**：缺失文件的卡片右键菜单显示"重新定位文件"，唤起系统文件选择器选择新路径
  - **MediaDetailDialog**：详情弹窗检测到文件缺失时，顶部显示警告横幅 + "重新定位"按钮
  - **LightBox 右键菜单**：大图浏览中右键"重新定位文件"（仅本地来源文件显示）
  - **图集详情页行模式右键菜单**：缺失文件可右键"重新定位文件"
- **批量重定向：** 设置页或图集右键菜单，批量更新某图集/文件夹的路径前缀

### 导入弹窗设计原则 [PRD §6.7]

- 所有弹窗复用同一套设计组件（标题栏、操作区、确认/取消按钮）
- 弹窗层级最小化，避免嵌套弹窗
- 保持轻量，减少性能占用

## API 端点

### POST `/api/media/scan` — 扫描路径

| 项 | 说明 |
|---|---|
| Body | `{paths[], recursive?}` |
| 返回 | `{results: [{path, total, existing}]}`，多路径时追加 `path="_total"` 汇总 |
| 用途 | 导入前预览，显示每个路径的媒体总数和已存在数 |

### POST `/api/media/import` — 导入媒体

| 项 | 说明 |
|---|---|
| Body | `{paths[], person_id?, album_id?, album_name?}` |
| 返回 | 导入 token（用于轮询进度） |

### GET `/api/media/import/{token}` — 导入进度

| 项 | 说明 |
|---|---|
| 参数 | 无 |
| 返回 | 当前导入进度信息 |

### POST `/api/media/import/{token}/cancel` — 取消导入

| 项 | 说明 |
|---|---|
| 参数 | 无 |
| 用途 | 取消进行中的导入任务，已导入记录保留 |

### POST `/api/media/import-clipboard` — 剪贴板导入

| 项 | 说明 |
|---|---|
| Body | FormData: `file` |
| 用途 | 粘贴导入单张图片，保存至 `AppData/imports/clipboard/` |

### POST `/api/media/upload-files` — 移动端文件上传导入

| 项 | 说明 |
|---|---|
| Body | FormData: `files[]`, `person_id?`, `album_id?` |
| 返回 | `{imported, media_ids[]}` |
| 用途 | 移动端文件上传，文件保存至 `AppData/imports/upload/` |

### POST `/api/media/list-files` — 文件列表与已导入状态

| 项 | 说明 |
|---|---|
| 返回 | 每个文件的 `existing: boolean` 状态 |
| 用途 | 逐个配置/子文件夹逐个视图中标记已导入文件 |

## 前端行为

### ImportDialog（导入向导）

**功能概述：**
- 文件选择 / 文件夹选择 / 子文件夹展平
- 每个子文件夹独立配置：图集名、所属人物
- 进度显示 + 可取消

**文件夹名智能拆分：**
- 若文件夹名包含 `_`、`-` 或空格，按第一个分隔符拆分，左侧作为默认人物名，右侧作为默认图集名（如 `Alice_Wedding` → 人物"Alice"，图集"Wedding"）
- 无分隔符时人物名和图集名均为完整文件夹名

**人物自动匹配与重名检测：**
- 拆分出的人物名若与数据库已有人物名匹配（不区分大小写），自动切换为"已有人物"模式
- 多个子文件夹拆分出相同人物名时，仅创建一次，后续复用同一人物 ID
- `resolvePersonAlbum` 在创建新人物前会通过 `findPersonByName()` 在已加载的 `persons` 列表中按名称（不区分大小写）查找匹配，匹配到则直接关联已有人物而非新建
- 所有新建人物输入框下方实时显示 `PersonMatchHint` 提示（"同名人物已存在，将自动关联"，amber 色）

**导入中保护：**
- `importing` 状态下，`onOpenChange` 忽略关闭请求
- `onInteractOutside` 和 `onEscapeKeyDown` 调用 `preventDefault()` 阻止对话框关闭

**导入前重复预览：**
- 选择来源后异步调用 `POST /api/media/scan` 扫描，显示每个文件夹的媒体总数/已存在数
- 子文件夹列表按"有新文件 → 全已存在"排序，全已存在行 `opacity-50`

**逐个配置已导入标记：**
- `POST /api/media/list-files` 返回每个文件的 `existing: boolean`
- 已导入文件显示"已导入"琥珀色标签（`text-amber-500`，置于文件名外层 flex 容器，`shrink-0` 防截断）
- 复选框 `disabled`，全选跳过已导入文件

**回收站重导入：**
- 后端 import 检测到同路径已软删除记录时，恢复 `is_deleted=False` 并更新关联，而非创建重复记录

**图集名去重：**
- 三种导入路径（逐个导入、子文件夹逐个、子文件夹批量）均使用 `createdAlbums` Map（key=`personId|albumName`）防止同名图集重复创建

**移动端文件上传：**
- 检测 `(pointer: coarse)` 时使用 HTML `<input type="file" multiple>` 替代 tkinter 文件选择器
- 文件通过 `POST /api/media/upload-files` 上传到服务器缓存（`AppData/imports/upload/`）
- 移动端隐藏文件夹选择和路径输入，不支持子文件夹模式

### ClipboardImportDialog（剪贴板导入）

- 捕获全局 `Ctrl+V` 粘贴事件
- 导入单张图片，通过 `POST /api/media/import-clipboard` 上传
- 文件保存至 `AppData/imports/clipboard/`
