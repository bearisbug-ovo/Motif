# 工作区与回收站

> 涉及代码：`backend/routers/workspace.py, recycle_bin.py` | `backend/models/workspace.py` | `frontend/src/pages/Workspace.tsx, RecycleBin.tsx` | `frontend/src/stores/workspace.ts`

## 需求摘要

**工作区** [PRD §5.9]：设置页「工作区」Tab 内的持久化图片暂存面板，上限 100 张，存储于 SQLite（对应 WorkspaceItem 表），刷新/关闭/跨设备均可访问。用途是为 AI 功能（换脸、局部修复等）提供跨人物/跨图集的参考图快速选取入口。

**回收站** [PRD §5.10]：设置页「回收站」Tab 内的软删除媒体恢复区。所有删除操作均为软删除（标记 `is_deleted = true`，记录 `deleted_at`），本地图任何情况下只删除记录、物理文件不动，生成图/截图软删除后进入回收站，永久删除时同时删除物理文件。默认 30 天后自动永久删除，保留天数可在设置页配置（设为 0 关闭自动清理）。[PRD §8.7]

## 数据模型（WorkspaceItem）

[PRD §3.9]

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| media_id | UUID | 关联媒体 ID（外键指向 Media） |
| added_at | datetime | 加入时间 |
| sort_order | int | 排序（支持拖拽调整） |

**说明：**
- 持久化存储于 SQLite，刷新/关闭不会丢失
- 上限 100 条，超出时提示用户清理
- 删除条目只解除引用关系，不影响源媒体

**ORM 字段** [开发指南 §4.6]：
```
id (UUID PK), media_id (FK→Media),
sort_order, created_at
```

## API 端点

| Method | 路径 | 说明 |
|---|---|---|
| GET | `/` | 工作区列表（含 Media 信息） |
| POST | `/` | 添加条目 Body: `{media_id}` |
| POST | `/batch` | 批量添加 Body: `{media_ids[]}` |
| DELETE | `/{item_id}` | 移除条目 |
| DELETE | `/` | 清空工作区 |
| PATCH | `/reorder` | 重排序 Body: `{ordered_ids[]}` |

| Method | 路径 | 说明 |
|---|---|---|
| GET | `/` | 列表（含保留倒计时） |
| POST | `/{mid}/restore` | 恢复 |
| DELETE | `/{mid}` | 永久删除 |

## 前端行为

### 工作区 [PRD §5.9]

- **位置**：设置页「工作区」Tab 内
- **容量**：上限 100 张，持久化存储于 SQLite，刷新/关闭/跨设备均可访问
- **添加方式**：任意图片卡片右键菜单"加入工作区"，或多选批量加入
- **工作区面板**：展示已加入的图片缩略图网格，点击图片进入 LightBox 大图浏览，右键菜单与图片卡片一致（AI 功能、加入工作区、移动到图集、资源管理器、评分、查看详情、删除），支持移除单张、拖拽排序（调用 `PATCH /api/workspace/reorder`）、一键清空
- **容量管理**：达到 100 张上限时提示用户"工作区已满，请先移除部分图片"，新增操作被拒绝

### 回收站 [PRD §5.10]

- **位置**：设置页「回收站」Tab 内
- **内容**：软删除的媒体记录（图片/视频）
- **展示**：只读浏览，显示原所属人物/图集信息、删除时间、距自动清理剩余天数
- **操作**：
  - 恢复到原位置（右键菜单）
  - 永久删除（右键菜单）：生成图同时删除物理文件，本地图只删除记录
  - 清空回收站（顶部按钮）
- **自动清理策略**：
  - 默认开启，保留 30 天后自动永久删除
  - 保留天数可在设置页配置（0 = 关闭自动清理，仅手动管理）
  - 自动清理在后端启动时和每日定时检查执行
  - 清理逻辑：`deleted_at + retention_days < now()` 的记录执行永久删除
- **无法在回收站内进行的操作**：评分、移动、AI 功能等

### 软删除与生成链 [PRD §8.7]

- 删除操作均为软删除（标记 `is_deleted = true`，记录 `deleted_at`）
- 本地图：任何情况下只删除记录，物理文件不动
- 生成图/截图：软删除后进入回收站；回收站永久删除时同时删除物理文件
- **生成链删除模式选择**：删除有子图的媒体时，弹出 DeleteChoiceDialog 让用户选择：
  - **级联删除（cascade）**：一并删除该图片的所有子图（默认）
  - **保留子图（reparent）**：仅删除该图片，其子图的 `parent_media_id` 指向被删图的父节点（即子图"上移"到祖父节点下）
  - 批量删除时，若选中项包含生成链图片，同样弹出选择对话框
  - 无子图的图片直接走简单确认对话框（ConfirmDialog）
