## 术语表

| 术语 | 含义 |
|------|------|
| 散图 | 不属于任何图集的媒体，直接归属于某个人物或无人物 |
| 生成图 | 由 AI 工作流产出的图片（source_type = `generated`），区别于用户导入的本地图 |
| 生成链 | 一张本地图通过多次 AI 操作产生的树形派生关系（本地图 → 生成图 → 二次生成图…） |
| 脱离链接 | 将生成图从生成链中摘出，转变为独立的本地图，断开与来源图的派生关系 |
| 工作区 | 跨人物/跨图集的临时参考图收藏夹，持久化存储于 SQLite |
| 人脸参考图 | 换脸、写真等功能中提供面部特征的参考图片 |
| 风格参考图 | 换脸功能中提供姿态/场景/风格的目标图片 |

## 版本规划

| 阶段 | 范围 | 交付标准 |
|------|------|----------|
| **P0 — 基础浏览与管理** | 媒体库浏览（人物/图集/大图模式）、导入流程、评分/筛选/排序、回收站、设置页、ComfyUI 连接管理、启动脚本 | 可完整导入本地图片并按人物/图集组织浏览，评分和筛选功能可用 |
| **P1 — 核心 AI 功能** | 高清放大、换脸（单张+批量）、局部修复（带提示词+自动反推）、任务队列（含 4 种启动模式）、工作区 | 可对图片执行 AI 操作并管理任务队列，生成链可视化可用 |
| **P2 — 高级 AI 功能** | 图生图、写真生成、动作库（DWPose 管理+动作组） | 图生图含提示词反推+润色完整链路可用，动作库支持批量提取和引用 |
| **P3 — 扩展工具** | 网页图片抓取器（5 平台适配）、平台账号管理 | 至少 2 个平台（小红书+通用网页）可用，其余平台可后续迭代 |

> 每个功能在下文中以 `[P0]` `[P1]` `[P2]` `[P3]` 标注所属阶段。

## 1. 项目概述

### 1.1 产品定位

Motif 是一款面向个人用户的本地图片/视频浏览、管理、AI 修复与联想生成工具。以 ComfyUI 作为 AI 能力后端，通过友好的 Web 界面提供照片整理、高清修复、换脸、写真生成等功能。

### 1.2 核心特性

- 以人物为核心的媒体管理体系
- 基于本地文件系统，不复制文件，只存储路径引用
- 通过 ComfyUI API 调用 AI 工作流，任务严格串行队列执行
- PWA + 响应式设计，支持 PC 与手机浏览器访问
- 支持多分辨率显示（1080p / 2K / 4K / 手机常用分辨率）
- 局域网多设备访问，FastAPI 绑定 `0.0.0.0`，无需认证

### 1.3 技术栈

| 层级 | 技术选型 |
|------|----------|
| 前端 | React + PWA，响应式布局 |
| 后端 | Python FastAPI（绑定 `0.0.0.0`，默认端口 `8000`） |
| 数据库 | SQLite（需配置 WAL 模式以支持多设备并发读取） |
| AI 后端 | ComfyUI（API 模式，`/prompt` 接口） |
| 启动方式 | 一键启动脚本（关闭旧进程 → 启动 FastAPI + ComfyUI → 前端轮询连接状态） |

## 2. 系统架构

### 2.1 整体架构

```
用户浏览器（PC / 手机）
      ↕ HTTP
FastAPI 后端
  ├── 文件系统管理（扫描、路径引用、缩略图）
  ├── SQLite 数据库（元数据 CRUD）
  ├── 任务队列管理（串行调度）
  └── ComfyUI 代理（/prompt API 调用）
      ↕ HTTP API
ComfyUI（本地运行）
  └── 各工作流节点
```

### 2.2 启动流程

1. 用户运行启动脚本
2. 脚本检测是否有旧的 Motif 进程（FastAPI、前端开发服务）和 ComfyUI 进程，有则关闭
3. 启动 FastAPI 后端
4. 检测 ComfyUI 是否已运行，未运行则使用配置的启动命令拉起 ComfyUI
5. 前端页面轮询 ComfyUI 连接状态，连接成功前 AI 功能按钮禁用，浏览功能不受影响
6. 顶部/侧边显示 ComfyUI 连接状态指示器

### 2.3 ComfyUI 连接状态

- **连接中**：旋转加载图标 + "ComfyUI 启动中..."
- **已连接**：绿色图标
- **连接失败 / 未连接**：红色图标 + 错误提示

**连接状态对功能的影响：**
- 浏览、管理、评分等功能不受影响
- AI 功能参数面板的"立即执行"按钮禁用，"加入队列"按钮保持可用
- 任务可以正常添加进队列，ComfyUI 恢复连接后按队列配置的启动模式继续执行

## 3. 数据模型

### 3.1 核心实体关系

```
人物 (Person)
  └──< 图集 (Album)
         └──< 媒体 (Media)  ←─────────────── 生成图也是 Media
                                              ↑
                                        关联人脸参考图
散图（无图集的媒体，直属人物）
无人物散图（导入时选择暂无人物）
```

### 3.2 Person（人物）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| name | string | 姓名 |
| cover_media_id | UUID? | 封面媒体 ID，默认第一张 |
| created_at | datetime | 创建时间 |
| avg_rating | float? | 平均评分（仅计算已评分媒体，全部未评分时为 null） |
| rated_count | int | 已评分媒体数量（用于前端显示"基于 N 张已评分"） |

### 3.3 Album（图集）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| person_id | UUID? | 所属人物，可为空（无人物图集） |
| name | string | 图集名称 |
| cover_media_id | UUID? | 封面，默认第一张 |
| created_at | datetime | 导入时间 |
| avg_rating | float? | 平均评分（仅计算已评分媒体，全部未评分时为 null） |
| rated_count | int | 已评分媒体数量 |
| is_generated_album | bool | 是否为 AI 生成图集（如批量换脸、写真套图） |
| source_face_media_id | UUID? | 生成图集关联的人脸参考图 |

### 3.4 Media（媒体）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| album_id | UUID? | 所属图集，可为空（散图） |
| person_id | UUID? | 所属人物（**约束：当 album_id 不为空时，必须等于 Album.person_id**） |
| file_path | string | 文件绝对路径 |
| media_type | enum | `image` / `video` |
| source_type | enum | `local`（本地图）/ `generated`（生成图）/ `screenshot`（视频截图） |
| parent_media_id | UUID? | 生成图/截图关联的来源媒体 ID（截图时指向父视频，通过父视频的 media_type=video 区分） |
| video_timestamp | float? | 截图对应视频时间戳（秒），仅 source_type=screenshot 时有值 |
| workflow_type | enum? | 使用的工作流类型 |
| generation_params | JSON? | 生成参数快照 |
| upscale_status | enum? | `pending` / `completed` / `failed` / `skipped`，仅生成图有值，标识默认高清放大状态 |
| rating | int? | 评分 1-5，null 表示未评分 |
| sort_order | int | 图集内排序 |
| imported_at | datetime | 导入/生成时间 |
| is_deleted | bool | 软删除标记 |
| deleted_at | datetime? | 删除时间 |
| thumbnail_path | string? | 缩略图缓存路径（可重建） |

**数据完整性约束：**
- `person_id` 一致性：当 Media 属于某 Album 时，Media.person_id 必须与 Album.person_id 相同。后端在导入和移动操作时强制校验。
- 生成链深度限制：`parent_media_id` 递归深度上限 10 层，超过时 UI 提示"已达最大嵌套深度"。

**说明：**
- 本地图的物理文件永远不删除，只删除数据库记录
- 生成图/截图进入回收站后永久删除时，同时删除物理文件
- 生成图可以多级嵌套（`parent_media_id` 指向另一张生成图）

### 3.5 Task（任务）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| workflow_type | enum | 工作流类型 |
| params | JSON | 工作流参数（含输入图路径） |
| status | enum | `pending` / `running` / `completed` / `failed` |
| queue_order | int | 队列排序，支持拖拽调整 |
| created_at | datetime | 创建时间 |
| started_at | datetime? | 开始时间 |
| completed_at | datetime? | 完成时间 |
| error_message | string? | 失败原因 |
| result_media_ids | JSON? | 生成结果的 Media ID 列表 |
| execution_mode | enum | `immediate`（立即执行）/ `queued`（加入队列） |

### 3.6 QueueConfig（队列配置，全局单例）

| 字段 | 类型 | 说明 |
|------|------|------|
| start_mode | enum | `manual`（手动）/ `auto`（自动）/ `cron`（Cron 定时）/ `delay`（延时） |
| cron_expression | string? | Cron 表达式（仅 `cron` 模式生效，如 `0 23 * * *`） |
| delay_minutes | int? | 延时分钟数（仅 `delay` 模式生效） |

**字段互斥规则：**
- `manual`：忽略 cron_expression 和 delay_minutes
- `auto`：忽略 cron_expression 和 delay_minutes
- `cron`：cron_expression 必填，忽略 delay_minutes
- `delay`：delay_minutes 必填，忽略 cron_expression

> 全局只有一条记录，控制整个任务队列的启动策略。

### 3.7 PoseAsset（动作资产）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| name | string | 动作名称 |
| tags | JSON | 标签列表（如["坐姿","户外","跟拍"]） |
| pose_image_path | string | DWPose 骨骼图路径 |
| created_at | datetime | 创建时间 |

**说明：**
- 用户上传原始参考图后，ComfyUI 提取 DWPose 骨骼图，仅保留骨骼图，原始图丢弃
- 骨骼图存入 `AppData/poses/` 目录
- 每个动作资产可被多个任务引用，任务记录中存储 `pose_asset_id`

### 3.8 PoseGroup（动作组）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| name | string | 动作组名称 |
| description | string? | 描述 |
| pose_ids | JSON | 有序的 PoseAsset ID 列表 |
| created_at | datetime | 创建时间 |

### 3.9 WorkspaceItem（工作区条目）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| media_id | UUID | 关联媒体 ID（外键指向 Media） |
| added_at | datetime | 加入时间 |
| sort_order | int | 排序（支持拖拽调整） |

**说明：**
- 持久化存储于 SQLite，刷新/关闭不会丢失
- 上限 100 条，超出时提示用户清理
- 删除条目只解除引用关系，不影响源媒体

### 3.10 PlatformAccount（平台账号）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| platform | enum | `xiaohongshu` / `bilibili` / `twitter` / `telegram` / `web` |
| username | string | 平台用户名（唯一标识） |
| display_name | string? | 显示名称 |
| person_id | UUID? | 关联人物（可为空，多账号可关联同一人物） |
| created_at | datetime | 首次创建时间 |

**说明：**
- 同一人物可关联多个平台账号（如小红书账号A + X账号B → 同一人物）
- 同一平台账号只能关联一个人物
- 账号与人物的关联关系在首次下载时询问，后续自动带入

### 3.11 DownloadRecord（下载记录）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| source_url | string | 原始链接（清洗后） |
| raw_text | string? | 原始粘贴文本（含分享提示文字） |
| platform | enum | 来源平台 |
| account_id | UUID? | 来源平台账号 |
| title | string? | 帖子/动态标题 |
| published_at | datetime? | 原帖发布日期 |
| media_count | int | 图片/视频数量 |
| album_id | UUID? | 生成的图集 ID |
| downloaded_at | datetime | 下载时间 |
| status | enum | `pending` / `completed` / `failed` |
| error_message | string? | 失败原因 |

### 3.12 数据库设计规范

**索引策略（SQLite）：**

| 表 | 索引 | 用途 |
|-----|------|------|
| Media | `(person_id, is_deleted)` | 人物主页查询 |
| Media | `(album_id, sort_order)` | 图集内排序展示 |
| Media | `(rating)` | 评分筛选/排序 |
| Media | `(parent_media_id)` | 生成链查询 |
| Media | `(is_deleted, deleted_at)` | 回收站列表及自动清理 |
| Task | `(status, queue_order)` | 队列调度 |
| WorkspaceItem | `(sort_order)` | 工作区展示 |
| DownloadRecord | `(status)` | 下载记录筛选 |

**并发策略：**
- SQLite 启用 WAL（Write-Ahead Logging）模式，支持多设备并发读取
- 写入操作由 FastAPI 后端统一管理，避免多客户端直接写入冲突
- 任务队列调度使用数据库乐观锁（`queue_order` + `status` 联合条件更新）

**Schema 迁移：**
- 使用 Alembic 管理数据库版本迁移
- 每次 schema 变更生成迁移脚本，启动时自动检测并执行
- `db/` 目录下保留 `alembic/versions/` 存放迁移历史

## 4. 目录结构

### 4.1 AppData 目录（可整体迁移，路径在设置中配置）

```
[AppData]/
  ├── db/
  │   └── main.sqlite
  ├── cache/
  │   └── thumbnails/          ← 缩略图缓存（可重建，不跟随迁移也无妨）
  ├── imports/
  │   └── clipboard/           ← 剪贴板导入图片的实际存储位置
  ├── generated/
  │   ├── upscale/             ← 高清放大结果
  │   ├── inpaint/             ← 局部修复/重绘结果
  │   ├── face_swap/           ← 换脸结果
  │   ├── portrait/            ← 写真生成结果
  │   └── screenshot/          ← 视频截图
  ├── poses/                   ← DWPose 骨骼图
  ├── workflows/               ← ComfyUI 工作流 JSON 模板
  └── downloads/               ← 网页抓取下载的图片/视频
      ├── xiaohongshu/
      ├── bilibili/
      ├── twitter/
      ├── telegram/
      └── web/
```

### 4.2 用户本地文件

任意位置的图片/视频文件夹，Motif 只存储绝对路径引用，不复制文件。

### 4.3 目录迁移

- 设置页提供"迁移 AppData 目录"功能
- 迁移时：复制整个 AppData 文件夹至新路径 → 更新 SQLite 中所有 `generated/` 和 `clipboard/` 的路径引用 → 更新设置中的 AppData 路径
- 用户本地图片的路径引用不受影响（文件未移动）

## 5. 各页面详细需求

### 5.1 左侧导航栏 `[P0]`

收起时显示图标，展开时显示图标 + 文字，默认收起。

```
[Motif Logo]
─────────────
📁 媒体库
📋 任务队列   ← 角标：左上旋转（运行中）/ 右上红色数字（失败）/ 右下绿色数字（完成，进入页面后清零）
🗂 工作区
🔧 小工具
🗑 回收站
⚙ 设置
```

### 5.2 媒体库 - 主页（人物一览） `[P0]`

**布局：** 网格卡片，每张卡片显示人物封面 + 姓名 + 平均评分（显示为"★4.2 (12)"格式，括号内为已评分数量；全部未评分时显示"未评分"）

**排序方式（可切换）：**
- 按导入时间（默认）
- 按平均评分（未评分人物排末尾；同分时按导入时间降序）
- 按名称

**筛选：** 评分筛选（等于 / 大于等于 / 小于等于指定星级）、媒体类型（图片 / 视频，可叠加）

**顶部操作：**
- 新建人物
- 导入图片（弹出导入流程）
- 随机探索（全局范围，见 8.8 节）

**人物卡片右键菜单：**
- 重命名
- 设为封面（从该人物图片中选）
- 删除人物（弹窗三选一）：
  - 仅删除人物，图片/图集变为无人物散图
  - 删除人物及图集记录，图片变为散图
  - 删除人物及所有内容（全部进回收站）

**悬浮操作：** 卡片右上角隐藏 `…` 按钮，悬浮 0.5s 后显示，点击触发同右键菜单

### 5.3 人物主页 `[P0]`

**布局：**
- 顶部：人物封面大图 + 姓名 + 平均评分
- 主体：图集网格列表（含生成图集）+ 散图区域并列展示

**图集卡片显示：** 封面 + 图集名 + 图片数量 + 平均评分 + 生成图集标记

**图集默认排序：** 导入时间，可切换（时间 / 评分 / 名称）

**筛选：** 与主页相同，筛选条件在所有层级生效（包括大图模式的上下张切换）

**顶部操作：**
- 导入图片（默认关联当前人物，可强制更改）
- 新建图集
- 随机探索（该人物范围，见 8.8 节）
- 多选模式入口

**图集卡片右键菜单：**
- 重命名
- 设为封面
- 移动到其他人物
- 为此图集应用换脸（批量换脸入口）
- 清理低分生成图（见 8.7 节）
- 删除图集（弹窗二选一）：
  - 仅删除图集，图片变为该人物散图
  - 删除图集及所有内容（进回收站）

**右键空白处触发规则（全局统一）：**
- 图集详情页空白处 → 触发图集右键菜单（含清理低分生成图）
- 人物主页空白处 → 触发人物层级上下文菜单（含清理低分生成图）
- 主页空白处 → 触发全局上下文菜单（含清理低分生成图）

### 5.4 图集详情页 `[P0]`

**布局：** 瀑布流 / 网格，顶部筛选和排序工具栏

**顶部操作：**
- 随机探索（该图集范围，见 8.8 节）
- 多选模式入口

**显示模式切换（筛选栏）：**
- 仅本地图（默认）
- 混合显示（本地图 + 生成图，生成图就近显示在来源图旁边）
- 仅生成图

**排序方式：** 导入时间 / 评分 / 名称，可切换

**视频卡片：** 右下角显示时长角标，左键单击进入视频播放大图模式

### 5.5 大图浏览模式 `[P0]`

**触发：** 点击任意图片卡片

**整体布局：**

```
│                              │                  │
├──────────────────────────────┴──────────────────┤
│        底部图集预览条（仅本地图，动态滚动高亮）        │

```

**顶部操作栏：** 与图片右键菜单保持功能一致，布局更合理

**底部预览条：**
- 仅显示同图集的本地图（不显示生成图）
- 当前筛选条件生效
- 动态滚动，当前图高亮居中
- 点击缩略图直接跳转（跳过生成图层级，直接切换图集图）

**右侧生成链面板（两种模式）：**

*简略模式（默认）：*
- 显示当前图在生成链中的位置（如"第 2 层 / 共 3 层"）
- 显示直接子生成图数量
- 生成图集以图集卡片显示，浏览到时展开

*详细树状图模式（按钮切换）：*
- 可视化展示完整生成链
- 用不同颜色/线型区分图集关系和生成关系
- 支持在生成链上操作脱离

**当前图是生成图时：** 右侧面板在该生成图所在的本地图+生成图预览列表中滑动高亮显示

**切换逻辑（主图区导航状态机）：**

当用户在主图区按 `→` 或右滑时，按以下优先级查找下一张（所有层级均受当前筛选条件约束）：

| 优先级 | 当前位置 | 下一张 | 终止条件 |
|--------|---------|--------|----------|
| 1 | 任意图 | 当前图的第一张直接子生成图 | 无子生成图时跳至优先级 2 |
| 2 | 生成图 | 同父图的下一张兄弟生成图 | 无更多兄弟时回到父图，跳至优先级 3 |
| 3 | 本地图 | 同图集内下一张本地图（按 sort_order） | 图集末尾时跳至优先级 4 |
| 4 | 图集末尾 | 同人物下一个图集的第一张本地图 | 人物下无更多图集时跳至优先级 5 |
| 5 | 人物末尾 | 下一个人物的第一张本地图 | 所有人物遍历完毕时提示"已到最后一张" |

按 `←` 或左滑时反向遍历，规则对称。

> 生成链递归最大深度 10 层，超过时跳过深层子节点。

**切换逻辑（指针在底部预览条时）：** 跳过生成图层级，直接切换图集图

- `←` / `→`：切换图片（遵循主图区切换逻辑）
- `Esc`：退出大图模式 / 退出沉浸模式
- `1`-`5`：快速评分
- 其他快捷键待细化

**鼠标操作：**
- 滚轮（主图区）：进入/调整放大镜模式倍数
- 右键：退出放大镜模式

#### 5.5.1 沉浸模式（全屏浏览）

**触发：** 顶部操作栏沉浸模式按钮，普通大图模式和随机探索均可开启

**开启后：** 隐藏所有 UI（顶部栏、底部预览条、右侧面板、左侧导航），仅保留图片全屏展示

**保留操作：**
- 切换图片：键盘 `←` / `→`；手机左右滑动
- 评分：键盘 `1`-`5`；手机**双击**图片弹出评分浮层（点击星级后浮层消失）

**退出沉浸模式：**
- PC：鼠标右键弹出菜单 → 选择"退出沉浸模式"；或按 `Esc`
- 手机：长按图片 800ms（手指不移动）弹出菜单 → 选择"退出沉浸模式"
- 手机长按与左右滑动互不干扰，滑动过程中不触发长按（以手指开始移动为判断依据，移动超过阈值即取消长按计时）

#### 5.5.2 放大镜模式

- **触发：** 左键单击主图区进入放大镜模式
- **行为：** 默认 2 倍放大，放大内容跟随指针移动，遮挡 UI（最高 z-index）
- **渲染方式：** 读取原始高分辨率图片对应区域裁剪展示（非 CSS scale，保证清晰度）
- **滚轮：** 调整放大倍数
- **退出：** 右键退出放大镜模式
- **超大图处理：** 不限制预览分辨率，完整加载原图以保证放大镜清晰度
- **禁用时机：** 蒙版编辑器中禁用放大镜模式

### 5.6 视频大图模式 `[P0]`

**布局：** 同图片大图模式，主区域替换为视频播放器

**播放器功能：** 播放/暂停、进度条、音量、全屏、逐帧步进（上一帧/下一帧按钮）

**截图操作：** 顶部操作栏提供"截取当前帧"按钮

- 截图作为子资产关联父视频（存储视频时间戳）

- 截图视为生成图，参与生成链管理
- 截图后提示用户"是否对此截图进行高清修复"（提供操作入口，不自动触发）

**视频封面：** 默认第 1 帧，有截图后使用第一张截图，可手动指定

### 5.8 任务队列页 `[P1]`

**布局：**
- 顶部：队列启动模式配置
- 主体：任务列表（可拖拽排序）

**队列启动模式（4 种，与 QueueConfig.start_mode 一一对应）：**
- **手动启动**（`manual`）：点击"开始执行"按钮触发
- **自动启动**（`auto`）：有待执行任务即自动开始
- **Cron 定时**（`cron`）：配置 Cron 表达式（如 `0 23 * * *` 表示每天 23:00）
- **延时启动**（`delay`）：配置 X 分钟后启动（从最后一个任务加入队列时刻起算）

**任务卡片显示：**
- 工作流类型、输入图预览、关键参数摘要
- 状态：待执行 / 执行中（进度动画）/ 已完成 / 失败（显示错误原因）
- 执行时间

**任务操作：**
- 拖拽调整队列顺序（仅对 `pending` 任务有效）
- 重新启动（适用于 `failed` 任务）
- 删除任务

**任务失败处理：** 失败后自动跳过，继续执行下一个任务，任务标记为 `failed`，需手动重启

**添加任务时：** 弹出选择"立即执行"或"加入队列"

**导航角标：**
- 左上：运行中旋转动画
- 右上：失败数量（红色），归零需手动清除失败任务
- 右下：完成数量（绿色），进入任务队列页后清零
- 三种角标可同时重叠显示

### 5.9 工作区 `[P1]`

**位置：** 左侧导航抽屉内

**容量：** 上限 100 张，**持久化存储于 SQLite**（对应 WorkspaceItem 表），刷新/关闭/跨设备均可访问

**用途：** 为 AI 功能（换脸、局部修复等）提供跨人物/跨图集的参考图快速选取入口

**添加方式：** 任意图片卡片右键菜单"加入工作区"，或多选批量加入

**工作区面板：** 展示已加入的图片缩略图网格，支持移除单张、拖拽排序、一键清空

**容量管理：** 达到 100 张上限时提示用户"工作区已满，请先移除部分图片"，新增操作被拒绝

### 5.10 回收站 `[P0]`

**位置：** 左侧导航独立入口

**内容：** 软删除的媒体记录（图片/视频）

**展示：** 只读浏览，显示原所属人物/图集信息、删除时间、距自动清理剩余天数

**操作：**
- 恢复到原位置（右键菜单）
- 永久删除（右键菜单）：生成图同时删除物理文件，本地图只删除记录
- 清空回收站（顶部按钮）

**自动清理策略：**
- 默认开启，保留 30 天后自动永久删除
- 保留天数可在设置页配置（0 = 关闭自动清理，仅手动管理）
- 自动清理在后端启动时和每日定时检查执行
- 清理逻辑：`deleted_at + retention_days < now()` 的记录执行永久删除

**无法在回收站内进行的操作：** 评分、移动、AI 功能等

## 6. 导入流程 `[P0]`

### 6.1 导入触发方式

- **粘贴导入：** 捕获全局剪贴板粘贴事件（`Ctrl+V`），导入单张图片，保存至 `AppData/imports/clipboard/`
- **文件选择：** 唤起系统文件选择器，支持选择单张/多张图片或文件夹

### 6.2 导入逻辑

**选择单张或多张图片：**
1. 询问关联人物（链接到已有人物 / 新建人物 / 暂时无人物）
2. 询问是否新建图集或暂时无图集

**选择无子文件夹的文件夹：**
1. 将该文件夹视为一个图集导入
2. 询问是否更改图集名称（默认使用文件夹名）
3. 询问关联人物

**选择有子文件夹的文件夹（递归处理）：**
1. 展开导入确认弹窗，列出所有子文件夹（多层嵌套全部展平）
2. 每行显示：文件夹路径 → 图集名称（可编辑）→ 所属人物（可选）
3. 用户逐行确认或批量设置后提交

### 6.3 导入进度与取消

- **进度显示：** 导入过程中显示进度条和计数（如"正在导入 45 / 230"）
- **可取消：** 导入过程中可随时取消，已导入的记录保留（不回滚）
- **大批量处理：** 超过 500 张图片时转为后台任务，顶部显示进度通知，用户可继续操作其他功能

### 6.4 人物上下文默认值

- 在**主页**触发导入：询问用户选择人物
- 在**人物主页**触发导入：默认关联当前人物，用户可强制更改

### 6.5 去重

- 以文件绝对路径作为唯一标识去重
- 重复扫描时跳过已存在记录，不重复导入

### 6.6 文件丢失处理

- 系统检测到文件路径不可访问时，卡片显示丢失标记（红色虚线边框 + 警告图标）
- 支持两种重定向方式：
  - **单张重定向：** 右键菜单"重新定位文件"，选择新路径
  - **批量重定向：** 设置页或图集右键菜单，批量更新某图集/文件夹的路径前缀

### 6.7 导入弹窗设计原则

- 所有弹窗复用同一套设计组件（标题栏、操作区、确认/取消按钮）
- 弹窗层级最小化，避免嵌套弹窗
- 保持轻量，减少性能占用

## 7. AI 功能详细需求

### 7.1 通用规范

- 所有 AI 功能通过底部抽屉式参数面板配置
- 参数面板底部固定"立即执行"和"加入队列"两个按钮
- 任务提交后，参数面板关闭，结果生成后自动存入 `AppData/generated/` 对应子目录
- 生成结果作为生成图关联至来源图（人脸参考图为准）
- **默认高清放大失败处理：** 放大步骤失败时保留原始生成图，标记 `upscale_status = failed`，在图片卡片上显示"放大失败"角标，用户可手动重试放大

### 7.2 高清放大 `[P1]`

**工作流：** 分块高清放大工作流（分块逻辑对用户透明，由 ComfyUI 节点自动处理）

**参数面板：**
- 放大倍数（2x / 4x，具体选项按工作流能力确定）
- 生成数量
- 立即执行 / 加入队列

**结果处理：** 存入 `generated/upscale/`，链接至原图

### 7.3 局部带提示词修复 `[P1]`

**工作流：** 指定位置带提示词修复工作流

**流程：** 触发 → 进入蒙版编辑器（带提示词模式）→ 绘制蒙版 → 配置参数 → 提交

**参数：**
- 提示词（中文，支持预设管理）
- 绘制强度
- 生成数量

**结果处理：** 存入 `generated/inpaint/`，链接至原图

### 7.4 局部自动反推修复 `[P1]`

**工作流：** 指定位置自动反推工作流 A / B（两个独立工作流，用户可感知并选择）

**流程：** 触发 → 进入蒙版编辑器（自动反推模式）→ 绘制蒙版 → 选择工作流 A 或 B → 配置参数 → 提交

**参数：**
- 工作流选择（A / B，展示各自适用场景说明）
- 绘制强度
- 生成数量

**结果处理：** 存入 `generated/inpaint/`，链接至原图

### 7.5 换脸 `[P1]`

**工作流：** qwen edit 换脸工作流

#### 单张换脸

**流程：**
1. 选中目标图（风格参考图）→ 触发"换脸"
2. 底部抽屉弹出
3. 选择人脸参考图（支持三种选择路径）：
   - 从工作区选取
   - 按人物 → 图集 → 图片层级浏览选取（可跨人物）
4. 配置生成数量
5. 立即执行 / 加入队列

**结果处理：** 存入 `generated/face_swap/`，**默认链接至人脸参考图**（非风格参考图），归属人脸参考图的人物。用户可在参数面板中更改归属人物。

#### 图集批量换脸

**触发：** 图集右键菜单"为此图集应用换脸"

**流程：**
1. 弹出确认弹窗，展示图集内所有本地图预览
2. 选择人脸参考图（同单张换脸选择方式）
3. 确认生成图集信息：
   - 图集名称（可编辑）
   - 所属人物（默认为人脸参考图所属人物，可更改为任意已有人物或新建人物）
4. 配置生成数量（每张图生成几次）
5. 立即执行 / 加入队列

**结果处理：** 生成图集存入 `generated/face_swap/`，图集整体关联至人脸参考图

### 7.6 图生图 `[P2]`

**工作流：** 独立图生图工作流

**触发入口：** 图片右键菜单 / 大图模式顶部操作栏

**操作流程：**
1. 触发后弹出底部抽屉参数面板
2. 参考图默认为当前选中图，可替换
3. 用户配置参数后提交任务

**参数面板布局：**

```
【参考图】
  当前图缩略图（可点击替换）

【提示词区】
  - 可编辑文本框（初始为空）
  - "分析并生成提示词"按钮：提交后在工作流中依次执行：
      ① ComfyUI 反推节点 → 生成基础描述（**具体节点待确认，见 12.2**）
      ② qwen-vl3 润色/场景转移 → 最终提示词
    （反推在任务执行时进行，非实时预览）
  - 场景模板快选（追加到提示词）

【动作参考（可选开关）】
  - 默认关闭
  - 开启后：选择单一动作（按标签筛选 / 随机抽取 1 个 / 按动作组选后指定）
  - 显示已选动作骨骼图预览
  - 启用动作参考时，qwen-vl3 润色会跳过动作描述（由 system prompt 控制）

【参数】
  - 图生图强度（denoising strength）
  - 生成数量
  - 默认高清放大：启用（默认）/ 临时关闭

【提交】
  - 立即执行 / 加入队列
```

**任务执行顺序（工作流内串联）：**
```
反推提示词 → qwen-vl3 润色 → 图生图生成 → 高清放大（如未关闭）→ 存储结果
```

**结果处理：**
- 存入 `generated/img2img/`
- 命名：`{参考图文件名}_img2img_{序号}.png`
- 关联至参考图（parent_media_id）
- 多张结果归入同一自动命名图集：`{参考图文件名}_img2img_{日期}`

**任务队列编辑：** 支持编辑所有创建时输入的参数（提示词、强度、数量、动作参考、放大开关）

## 8. 图片管理功能

### 8.1 评分系统

- 评分范围：1-5 星，可清除（清除后视为未评分）
- **平均分计算规则：仅计算已评分媒体的平均值，未评分媒体不参与计算**
- 全部未评分时，avg_rating 为 null，UI 显示"未评分"
- 前端显示格式：`★4.2 (12)` — 平均分 + 已评分数量
- 图集和人物的平均评分实时计算（基于下属已评分媒体）
- 评分入口：大图模式顶部、图片右键菜单、多选批量评分

### 8.2 筛选系统

筛选条件可叠加，在所有层级生效（包括大图模式的上下张切换）：

| 维度 | 选项 |
|------|------|
| 评分 | 等于 / 大于等于 / 小于等于 指定星级 |
| 媒体类型 | 图片 / 视频（可多选叠加） |
| 来源类型 | 本地图 / 生成图（在混合显示模式下） |

### 8.3 排序系统

| 层级 | 排序选项 |
|------|----------|
| 人物列表 | 导入时间 / 平均评分 / 名称 |
| 图集列表 | 导入时间 / 平均评分 / 名称 |
| 图集内图片 | 导入时间 / 评分 / 名称 |

**评分排序规则：** 有评分的按平均评分降序排列，未评分（avg_rating = null）的排末尾；未评分组内按导入时间降序

### 8.4 封面管理

- 人物封面：默认第一张图，右键图片"设为人物封面"
- 图集封面：默认第一张图，右键图片"设为图集封面"
- 视频封面：默认第 1 帧；有截图后使用第一张截图；多张截图时可手动指定

### 8.5 生成链管理

- 生成图支持多级嵌套（生成图的生成图）
- 任意生成图可脱离链接变为本地图（子数据：评分、二次生成图全部跟随）
- 脱离时物理文件不移动，数据库中解除关联关系，source_type 改为 `local`

### 8.6 软删除与回收站

- 删除操作均为软删除（标记 `is_deleted = true`，记录 `deleted_at`）
- 本地图：任何情况下只删除记录，物理文件不动
- 生成图/截图：软删除后进入回收站；回收站永久删除时同时删除物理文件
- 回收站支持恢复到原位置
- 回收站手动清空
- **自动清理：默认 30 天后自动永久删除，保留天数可在设置页配置（设为 0 关闭自动清理）**

### 8.7 批量清理低分生成图

**功能：** 批量删除当前作用域内评分低于指定值的生成图

**作用域：**
- 主页（全局）：清理所有人物下的低分生成图
- 人物主页：清理该人物下的低分生成图
- 图集详情页：清理该图集下的低分生成图

**触发入口：** 各层级右键空白处菜单

**清理规则：**
- 仅清理生成图（source_type 为 `generated` 或 `screenshot`），本地图不受影响
- **未评分的生成图永远不参与清理**
- 评分严格小于指定阈值的生成图将被删除

**操作流程：**
1. 弹出阈值设置面板（选择评分阈值，如"清理低于 3 星的生成图"）
2. 展示预览列表：将被删除的生成图缩略图及总数量
3. 用户二次确认后执行软删除（进回收站）

### 8.8 随机探索

**功能：** 在指定作用域和筛选条件内随机排列媒体，以大图模式浏览

**触发入口：** 各层级顶部工具栏"随机探索"按钮，自动继承当前作用域和已设筛选条件

**作用域：**
- 从触发层级自动继承（主页 = 全局，人物主页 = 该人物，图集页 = 该图集）
- 进入后可手动切换作用域（全局 / 指定人物 / 指定图集）

**筛选选项（可叠加）：**
- 评分：未评分 / 大于等于指定星级
- 媒体类型：图片 / 视频
- 来源类型：本地图 / 生成图

**探索行为：**
- 进入时对符合条件的媒体随机洗牌，生成本次探索序列
- 筛选条件变更后需手动点击"重新洗牌"，不动态应用
- 支持在序列内前进 / 后退切换
- 评分操作实时生效，但不重排本次序列

**界面：** 复用大图模式，顶部额外显示"探索进度"（如 12 / 87）和"重新洗牌"按钮

**沉浸模式：** 可在随机探索中开启沉浸模式（见 5.5.1 节）

### 9.1 右键菜单 vs 悬浮菜单

- 从屏幕底部滑入

- 复用同一套弹窗组件
- 避免嵌套弹窗

- 加载策略：懒加载（首次进入视口时生成）

  - `≥ 1920px`：大屏 PC / 4K（更多列，更大缩略图）
- 手机端蒙版画笔支持触屏绘制（单指绘制 + 双指缩放）
- 手机端主要场景：浏览、评分筛选、任务管理
- 手机端 AI 功能完整保留，但蒙版编辑精度有限
- 放大镜模式仅 PC 端支持，手机端使用双指缩放替代

### 9.7 键盘快捷键（大图模式）

## 10. 技术实现要点

### 10.1 工作流类型与命名规范

**workflow_type 枚举及文件名缩写：**

| 功能 | workflow_type | 文件名缩写 |
|------|--------------|-----------|
| 高清放大 | upscale | upscale |
| 局部带提示词修复 | inpaint_prompt | inpaint_p |
| 局部自动反推修复 A | inpaint_auto_a | inpaint_a |
| 局部自动反推修复 B | inpaint_auto_b | inpaint_b |
| 换脸 | face_swap | faceswap |
| 图生图 | img2img | img2img |
| 写真生成 | portrait | portrait |
| DWPose 提取 | pose_extract | pose |

**文件命名规则（全局统一）：**
```
{参考图文件名}_{workflow_type缩写}_{序号}.png
示例：IMG_001_img2img_001.png
     IMG_001_faceswap_002.png
```

**默认高清放大规则：**
- 除高清放大本身外，所有 AI 功能完成后自动串联一次高清放大
- 放大成功时：最终存储放大后的图，中间未放大的原始生成图丢弃
- **放大失败时：保留原始生成图，标记 `upscale_status = failed`，图片卡片显示"放大失败"角标，用户可从右键菜单手动重试放大**
- 默认放大倍数在设置页配置全局默认值
- 用户可在单次任务参数面板底部临时关闭本次放大（标记 `upscale_status = skipped`）
- 命名规则中不体现放大步骤（命名以主工作流类型为准）

### 10.2 ComfyUI 集成

- 通过 `/prompt` 接口提交工作流 JSON
- 通过 `/history` 接口轮询任务状态（轮询间隔 2 秒）
- 工作流 JSON 模板存储在 `AppData/workflows/` 目录下，每个工作流类型一个 JSON 文件
- 参数占位符格式：`{{param_name}}`，如 `{{input_image_path}}`、`{{denoise_strength}}`
- 后端在提交前将占位符替换为实际参数值
- 任务严格串行，FastAPI 维护队列，一次只向 ComfyUI 提交一个任务
- 单任务超时阈值：默认 10 分钟，可在设置页配置

### 10.3 任务队列

- SQLite 存储任务记录和队列顺序
- FastAPI 后台线程维护队列调度
- 支持 4 种启动模式：手动（manual）/ 自动（auto）/ Cron 定时（cron）/ 延时（delay）
- 延时模式从最后一个任务加入队列时刻起算
- 任务失败自动跳过，继续执行下一个
- 失败任务保留错误信息，支持手动重试

### 10.4 文件系统监控

- 导入时记录文件绝对路径，不复制文件
- 访问文件时检测路径可用性，不可用时显示丢失标记
- 剪贴板导入图片保存至 `AppData/imports/clipboard/`

### 10.5 放大镜实现

- 读取原始高分辨率图片，按指针位置裁剪对应区域渲染
- 放大层使用最高 z-index，覆盖所有 UI 元素
- 蒙版编辑器中禁用放大镜，改为画布整体缩放（滚轮缩放以指针为中心）

### 10.6 蒙版编辑器

- 使用 HTML5 Canvas 实现画笔绘制
- 蒙版以绿色半透明层叠加在图片上
- 支持撤销/重做（操作历史栈）
- 蒙版导出为灰度图传入 ComfyUI 工作流

### 10.7 视频处理

- HTML5 原生视频播放器，支持局域网串流
- 截图使用 Canvas 从视频元素捕获当前帧
- 提供逐帧步进功能（前进/后退 1 帧）

### 10.8 生成链数据结构

- `parent_media_id` 自引用外键实现多级嵌套
- **最大递归深度 10 层**，后端在创建生成图时检查深度，达到上限时拒绝并提示"已达最大嵌套深度"
- 树状图渲染需递归查询子节点（后端提供 `/api/media/{id}/tree` 接口返回完整生成链）
- 脱离操作：更新 `source_type` 为 `local`、清空 `parent_media_id` 和 `workflow_type`

## 11. 小工具

### 11.1 动作库 `[P2]`

**功能：** 管理 DWPose 动作资产，供图生图和写真生成任务使用

**入口：** 小工具页面独立模块

#### 11.1.1 动作库主页

**布局：** 网格卡片，每张卡片显示骨骼图预览 + 名称 + 标签

**待处理提示：** 主页顶部醒目显示"N 个动作待命名/打标签"，点击进入批量编辑界面

**未命名条目样式：** 混入列表但以特殊样式标记（灰色边框 + 警告图标），支持批量选中后统一打标签，名称各自单独填写

**筛选：** 按标签筛选，按名称搜索

**顶部操作：**
- 上传参考图（触发 DWPose 提取流程，见 11.1.2）
- 从媒体库选取（选择已入库的图片或图集生成骨骼图，见 11.1.2）
- 新建动作组

**动作卡片右键菜单：**
- 编辑名称/标签
- 加入动作组
- 查看使用此动作的全部生成图（见 11.1.3）
- 删除

#### 11.1.2 上传与提取流程

**来源方式一：上传本地文件**

- 单张图片：弹出命名+标签输入框，填写后提交任务（名称标签随任务传入，提取完成后直接入库）
- 文件夹：递归搜索文件夹内所有图片 → 展示预览确认列表（文件名 + 缩略图）→ 用户确认后批量加入队列

**来源方式二：从媒体库选取**

- 支持选择单张图片、多张图片或整个图集
- 选定后展示预览确认列表，确认后批量加入队列
- 单张时同样可以直接填写名称+标签随任务传入

**提取任务行为：**
- 每张图片作为独立任务加入公共串行任务队列（与其他 AI 任务混排，队列页显示任务类型为"DWPose 提取"）
- 提取失败的任务自动跳过，继续执行下一个
- 提取完成后仅保留骨骼图，原始参考图丢弃（媒体库来源的原图不受影响）
- 骨骼图存入 `AppData/poses/`
- 批量提取完成的条目在动作库显示为"待命名/待标签"状态

#### 11.1.3 动作使用记录视图

**触发：** 动作卡片右键菜单"查看使用此动作的全部生成图"

**展示：** 聚合展示所有使用该动作生成的图片（跨人物/跨图集），支持大图模式的全部功能（评分、AI 操作、生成链查看等）

#### 11.1.4 动作组管理

**功能：** 将多个动作资产编组，供写真生成批量使用

**动作组详情页：**
- 动作列表（有序，可拖拽排序）
- 添加/移除动作
- 典型数量：9 个动作

### 11.2 网页图片抓取器 `[P3]`

**功能：** 解析社交平台帖子或含图网页链接，下载图片/视频并导入媒体库

**支持平台：**

| 平台 | 技术方案 | 授权方式 |
|------|----------|----------|
| 小红书 | 需登录态，图片有防盗链 | 账号 Cookie |
| B站动态/专栏 | API 较开放 | 账号 Cookie（部分内容需要） |
| X / Twitter | API 限制严格 | 账号 Cookie 或 API Key |
| Telegram | 公开频道直接抓取，私有频道需授权 | Telethon 账号授权 |
| 通用网页 | requests + Playwright 渲染动态页 | 无需授权 |

**各平台授权配置：** 统一在设置页"平台授权"分区配置 Cookie / API Key / Telethon 授权

#### 11.2.1 触发方式

**PC 端：**
- 在小工具页面的输入框粘贴链接，点击"解析"按钮

**手机端：**
- 同上，但粘贴内容可能包含分享提示文字（如"博主名发布了一条小红书笔记，快来看看吧~ http://xhslink.com/xxxxx"）
- 后端自动从粘贴文本中提取有效链接，支持多种平台的短链和完整链接格式

#### 11.2.2 下载流程

1. **粘贴文本 → 提取链接**
   - 后端从原始文本中正则提取有效 URL
   - 识别平台类型
   - 检查对应平台授权是否配置，未配置则提示前往设置

2. **解析元数据**（后台请求，前端显示加载状态）
   - 博主用户名 / 显示名
   - 帖子标题
   - 发布日期
   - 图片/视频预览（缩略图列表）
   - 媒体数量

3. **弹窗展示解析结果，用户确认**

4. **账号关联步骤**
   - 检查该平台账号是否已有关联人物记录
   - **已关联**：自动带入关联人物，用户可更改
   - **未关联**：询问关联到哪个人物（从已有人物选取 / 新建人物 / 暂不关联）
   - 提供"记住此账号的关联关系"选项（勾选后写入 PlatformAccount 表）

5. **图集配置**
   - 图集名称默认为帖子标题，可编辑
   - 选择归入方式：
     - 新建图集（默认）
     - 合并到已有图集（从该人物图集中选）
     - 作为散图（不归入图集）
   - 同一账号再次下载时：默认新建图集，可选已有图集

6. **确认下载**
   - 图片存入 `AppData/downloads/{platform}/`
   - 视频同样下载
   - 走正常导入流程，写入 Media 记录和 DownloadRecord 记录
   - 下载完成后跳转至生成的图集

#### 11.2.3 下载记录页

- 展示历史下载记录列表（来源平台、账号、标题、日期、状态）
- 失败记录显示错误原因，支持重试
- 点击记录可跳转至对应图集

#### 11.2.4 平台账号管理

**入口：** 设置页"平台账号"分区

**功能：**
- 查看所有已识别的平台账号列表
- 编辑账号与人物的关联关系
- 删除账号记录（不影响已下载的图片）

## 13. 异常处理规范

### 13.1 ComfyUI 异常

| 场景 | 预期行为 |
|------|----------|
| ComfyUI 进程未启动 | 前端显示"ComfyUI 启动中..."，AI 功能按钮禁用（"加入队列"仍可用），每 5 秒轮询连接状态 |
| ComfyUI 进程崩溃 | 后端检测到连接断开后标记状态为"连接失败"，当前运行中的任务标记为 `failed`（error_message = "ComfyUI 连接中断"），后端每 10 秒尝试重连，连续失败 30 次后停止重试并提示用户手动检查 |
| ComfyUI 任务超时 | 单任务超时阈值 10 分钟（可配置），超时后标记 `failed`，跳至下一任务 |

### 13.2 存储异常

| 场景 | 预期行为 |
|------|----------|
| 磁盘空间不足 | 生成任务执行前检查剩余空间（阈值 500MB），空间不足时拒绝执行并提示"磁盘空间不足，请清理后重试" |
| 文件路径不可访问 | 卡片显示丢失标记（红色虚线边框 + 警告图标），不阻塞其他功能 |
| 缩略图生成失败 | 显示默认占位图，不影响其他操作，后台重试一次 |

### 13.3 网络异常（局域网场景）

| 场景 | 预期行为 |
|------|----------|
| 客户端与后端断连 | 前端显示"连接中断，正在重连..."横幅，每 3 秒重试，恢复后自动刷新当前页面数据 |
| 请求超时 | 前端操作 10 秒无响应显示"操作超时"提示，用户可点击重试 |

### 13.4 外部平台异常（网页抓取器） `[P3]`

| 场景 | 预期行为 |
|------|----------|
| Cookie 过期 | 提示"授权已过期，请在设置页更新 Cookie" |
| 平台限流/封禁 | 显示具体错误信息，建议等待后重试，记录失败到 DownloadRecord |
| 内容已删除/不可访问 | 提示"原始内容不可访问"，记录失败原因 |

## 14. 性能指标

| 场景 | 目标 | 测试条件 |
|------|------|----------|
| 人物列表页首屏渲染 | < 500ms | 100 个人物，PC Chrome |
| 图集列表页首屏渲染 | < 500ms | 单人物 50 个图集，PC Chrome |
| 图集内图片加载（缩略图） | < 1s（首屏） | 1000 张图的图集，PC Chrome |
| 图集内图片加载（手机） | < 2s（首屏） | 1000 张图的图集，手机 Safari |
| 大图切换响应 | < 100ms | 切换到已缓存的相邻图片 |
| SQLite 查询（筛选+排序） | < 200ms | 10000 条 Media 记录 |
| 缩略图缓存最大磁盘占用 | 可配置，默认 2GB | 超出时按 LRU 策略清理最旧缩略图 |

| 生成链最大递归深度 | 10 层 | 超出时 UI 提示，不崩溃 |

## 15. 核心 API 端点（概要）

> 详细 API 文档建议使用 OpenAPI/Swagger 自动生成，以下为核心端点概要。

### 15.1 媒体管理

| Method | 端点 | 说明 |
|--------|------|------|
| GET | `/api/persons` | 人物列表（支持 sort、filter 参数） |
| POST | `/api/persons` | 创建人物 |
| GET | `/api/persons/{id}/albums` | 人物下的图集列表 |
| GET | `/api/albums/{id}/media` | 图集内媒体列表（支持 sort、filter、source_type 参数） |
| POST | `/api/import` | 导入媒体（Body: paths、person_id、album_id） |
| PATCH | `/api/media/{id}` | 更新媒体信息（评分、移动图集等） |
| PATCH | `/api/media/batch` | 批量更新（批量评分、批量移动） |
| DELETE | `/api/media/{id}` | 软删除 |

### 15.2 AI 任务

| Method | 端点 | 说明 |
|--------|------|------|
| POST | `/api/tasks` | 创建任务（Body: workflow_type、params、execution_mode） |
| GET | `/api/tasks` | 任务列表（支持 status 筛选） |
| PATCH | `/api/tasks/{id}` | 更新任务（编辑参数、调整队列顺序） |
| POST | `/api/tasks/{id}/retry` | 重试失败任务 |
| DELETE | `/api/tasks/{id}` | 删除任务 |
| POST | `/api/queue/start` | 手动触发队列执行 |
| GET | `/api/queue/config` | 获取队列配置 |
| PUT | `/api/queue/config` | 更新队列配置 |

### 15.3 工作区

| Method | 端点 | 说明 |
|--------|------|------|
| GET | `/api/workspace` | 获取工作区条目列表 |
| POST | `/api/workspace` | 添加条目（Body: media_id） |
| DELETE | `/api/workspace/{id}` | 移除条目 |
| DELETE | `/api/workspace` | 清空工作区 |

### 15.4 系统

| Method | 端点 | 说明 |
|--------|------|------|
| GET | `/api/system/status` | 系统状态（ComfyUI 连接状态、磁盘空间、队列状态） |
| GET | `/api/system/config` | 获取设置 |
| PUT | `/api/system/config` | 更新设置 |
| POST | `/api/system/migrate-appdata` | 触发 AppData 迁移 |
| GET | `/api/recycle-bin` | 回收站列表 |
| POST | `/api/recycle-bin/{id}/restore` | 恢复 |
| DELETE | `/api/recycle-bin/{id}` | 永久删除 |
| DELETE | `/api/recycle-bin` | 清空回收站 |

### 15.5 网页抓取 `[P3]`

| Method | 端点 | 说明 |
|--------|------|------|
| POST | `/api/download/parse` | 解析链接，返回元数据预览 |

| GET | `/api/download/records` | 下载记录列表 |
