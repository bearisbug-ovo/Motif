## 2. 功能描述模糊

### 2.1 蒙版编辑器的模式对应关系（§5.7 + §7.3 + §7.4）

**问题**：PRD 描述了"带提示词"和"自动反推"两个大类模式，但"自动反推"又分 A/B 两个工作流。实际实现为三个并列模式（Flux / SDXL / Klein），用户在一个下拉框中选择，没有二级分类。

**建议更新 PRD §5.7 底部提交区**：

```
模式选择（三选一）：
- Flux（带提示词）：需输入提示词，绘制强度可调
- SDXL（通用 A）：无需提示词，绘制强度可调
- Klein（通用 B）：无需提示词，无强度参数
```

### 2.2 生成数量参数（§7.2 - §7.5）

**问题**：PRD 所有 AI 功能均提到"生成数量"参数，但当前实现中**没有生成数量控制**，每次固定生成 1 张。

**原因**：ComfyUI 单次任务生成多张需要工作流内循环或多次提交，实现复杂度高，且串行队列下多次提交更简单（用户可手动多次提交同一任务）。

**建议**：
- 选项 A：保留 PRD 描述，后续实现（前端重复提交 N 次相同任务）
- 选项 B：删除"生成数量"，简化为"每次生成 1 张，需多张可重复提交"

### 2.3 蒙版导出格式（§10.6）

| PRD | 实际实现 |
|---|---|
| "蒙版导出为**灰度图**传入 ComfyUI 工作流" | RGBA PNG（alpha=0 为修复区域，alpha=255 为保留区域） |

**建议更新 PRD**：RGBA PNG 格式兼容 ComfyUI 的 `LoadImage` 节点（自动提取 alpha 通道作为 mask），且保留了与底图的尺寸对应关系。

### 2.4 图集详情页布局描述（§5.4）

| PRD | 实际实现 |
|---|---|
| "瀑布流 / 网格" | 等高行布局（flex row）/ 方块网格（grid） |

**建议更新 PRD**：当前实现的"等高行布局"更接近 Google Photos 风格，非传统瀑布流。

### 2.5 图集详情显示模式（§5.4）

PRD 描述了三种显示模式：
1. 仅本地图（默认）
2. 混合显示（本地图 + 生成图，生成图就近显示在来源图旁边）
3. 仅生成图

实际实现为 `source_type` 筛选器（all / local / generated），功能等价但**缺少"就近显示"逻辑**（生成图未自动排列在来源图旁边）。

**建议**：标记"就近显示"为 P1 后续优化项，当前实现满足基本筛选需求。

## 5. Cron 模式实现不完整

PRD §5.8 / §10.3 描述了 Cron 表达式支持（如 `0 23 * * *`），但当前实现仅做了简单的 60 秒轮询，**未解析 cron 表达式**。

**建议**：
- 安装 `croniter` 库，在 `_should_execute()` 中正确判断 cron 触发时间
- 或将 cron 模式标记为 P2 功能
