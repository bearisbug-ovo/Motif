## 术语表

| 术语 | 含义 |
|------|------|
| 散图 | 不属于任何图集的媒体，直接归属于某个人物或无人物 |
| 生成图 | 由 AI 工作流产出的图片（source_type = `generated`），区别于用户导入的本地图 |
| 生成链 | 一张本地图通过多次 AI 操作产生的树形派生关系（本地图 → 生成图 → 二次生成图…） |
| 脱离链接 | 将生成图从生成链中摘出，转变为独立的本地图，断开与来源图的派生关系 |
| 工作区 | 跨人物/跨图集的临时参考图收藏夹，持久化存储于 SQLite |
| 人脸参考图 | 换脸、写真等功能中提供面部特征的参考图片 |
| 风格参考图 | 换脸功能中提供姿态/场景/风格的目标图片 |

## 1. 项目概述

### 1.1 产品定位

Motif 是一款面向个人用户的本地图片/视频浏览、管理、AI 修复与联想生成工具。以 ComfyUI 作为 AI 能力后端，通过友好的 Web 界面提供照片整理、高清修复、换脸、写真生成等功能。

### 1.2 核心特性

- 以人物为核心的媒体管理体系
- 基于本地文件系统，不复制文件，只存储路径引用
- 通过 ComfyUI API 调用 AI 工作流，任务严格串行队列执行
- PWA + 响应式设计，支持 PC 与手机浏览器访问
- 支持多分辨率显示（1080p / 2K / 4K / 手机常用分辨率）
- 局域网多设备访问，FastAPI 绑定 `0.0.0.0`，无需认证

### 1.3 技术栈

| 层级 | 技术选型 |
|------|----------|
| 前端 | React + PWA，响应式布局 |
| 后端 | Python FastAPI（绑定 `0.0.0.0`，默认端口 `8000`） |
| 数据库 | SQLite（需配置 WAL 模式以支持多设备并发读取） |
| AI 后端 | ComfyUI（API 模式，`/prompt` 接口） |
| 启动方式 | 一键启动脚本（关闭旧进程 → 启动 FastAPI + ComfyUI → 前端轮询连接状态） |

## 2. 系统架构

### 2.1 整体架构

```
用户浏览器（PC / 手机）
      ↕ HTTP
FastAPI 后端
  ├── 文件系统管理（扫描、路径引用、缩略图）
  ├── SQLite 数据库（元数据 CRUD）
  ├── 任务队列管理（串行调度）
  └── ComfyUI 代理（/prompt API 调用）
      ↕ HTTP API
ComfyUI（本地运行）
  └── 各工作流节点
```

### 2.2 启动流程

用户双击 `start.bat`（调用 `start.ps1`），启动器自动完成以下步骤：

1. **单实例检测**：通过命名 Mutex 防止重复启动，已有实例时提示并退出
2. **清理旧进程**：检测并关闭占用端口 8000（后端）、5173（前端）的旧进程
3. **启动后端**：以隐藏窗口启动 `uvicorn`，日志写入 `.logs/backend.log`，等待端口 8000 就绪（超时 30 秒）
4. **启动前端**：以隐藏窗口启动 `npm run dev`，日志写入 `.logs/frontend.log`，等待端口 5173 就绪（超时 20 秒）
5. **启动 ComfyUI**：检测 ComfyUI 是否已运行（请求 `/object_info/KSampler`），未运行则拉起（独立进程，不随启动器退出）
6. **打开浏览器**：自动打开 `http://localhost:5173`
7. **进入交互模式**：显示服务状态和可用命令，等待用户输入

#### 交互命令

| 按键 | 行为 |
|------|------|
| `r` / `R` | 快速重启后端 + 前端（不重启 ComfyUI） |
| `Enter`（或其他任意输入） | 停止所有服务并退出启动器 |

#### 进程管理

- 后端和前端进程通过 Win32 Job Object 绑定到启动器，启动器退出时自动终止所有子进程
- ComfyUI 作为独立进程启动，不受启动器生命周期影响
- 所有日志输出到项目根目录 `.logs/` 下：`backend.log`、`backend-error.log`、`frontend.log`、`frontend-error.log`

#### 前端 ComfyUI 状态

- 前端页面轮询 ComfyUI 连接状态，连接成功前 AI 功能按钮禁用，浏览功能不受影响
- 顶部/侧边显示 ComfyUI 连接状态指示器

### 2.3 ComfyUI 连接状态

- **连接中**：旋转加载图标 + "ComfyUI 启动中..."
- **已连接**：绿色图标
- **连接失败 / 未连接**：红色图标 + 错误提示

**连接状态对功能的影响：**
- 浏览、管理、评分等功能不受影响
- AI 功能参数面板的"立即执行"按钮禁用，"加入队列"按钮保持可用
- 任务可以正常添加进队列，ComfyUI 恢复连接后按队列配置的启动模式继续执行

## 3. 数据模型

### 3.1 核心实体关系

```
人物 (Person)
  └──< 图集 (Album)
         └──< 媒体 (Media)  ←─────────────── 生成图也是 Media
                                              ↑
                                        关联人脸参考图
散图（无图集的媒体，直属人物）
无人物散图（导入时选择暂无人物）
```

### 3.2 Person（人物）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| name | string | 姓名 |
| cover_media_id | UUID? | 封面媒体 ID，默认第一张 |
| created_at | datetime | 创建时间 |
| avg_rating | float? | 平均评分（仅计算已评分媒体，全部未评分时为 null） |
| rated_count | int | 已评分媒体数量（用于前端显示"基于 N 张已评分"） |

### 3.3 Album（图集）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| person_id | UUID? | 所属人物，可为空（无人物图集） |
| name | string | 图集名称 |
| cover_media_id | UUID? | 封面，默认第一张 |
| created_at | datetime | 导入时间 |
| avg_rating | float? | 平均评分（仅计算已评分媒体，全部未评分时为 null） |
| rated_count | int | 已评分媒体数量 |
| is_generated_album | bool | 是否为 AI 生成图集（如批量换脸、写真套图） |
| source_face_media_id | UUID? | 生成图集关联的人脸参考图 |

### 3.4 Media（媒体）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| album_id | UUID? | 所属图集，可为空（散图） |
| person_id | UUID? | 所属人物（**约束：当 album_id 不为空时，必须等于 Album.person_id**） |
| file_path | string | 文件绝对路径 |
| media_type | enum | `image` / `video` |
| source_type | enum | `local`（本地图）/ `generated`（生成图）/ `screenshot`（视频截图） |
| parent_media_id | UUID? | 生成图/截图关联的来源媒体 ID（截图时指向父视频，通过父视频的 media_type=video 区分） |
| video_timestamp | float? | 截图对应视频时间戳（秒），仅 source_type=screenshot 时有值 |
| workflow_type | enum? | 使用的工作流类型 |
| generation_params | JSON? | 生成参数快照 |
| rating | int? | 评分 1-5，null 表示未评分 |
| sort_order | int | 图集内排序 |
| imported_at | datetime | 导入/生成时间 |
| is_deleted | bool | 软删除标记 |
| deleted_at | datetime? | 删除时间 |
| thumbnail_path | string? | 缩略图缓存路径（可重建） |

**数据完整性约束：**
- `person_id` 一致性：当 Media 属于某 Album 时，Media.person_id 必须与 Album.person_id 相同。后端在导入和移动操作时强制校验。
- 生成链深度限制：`parent_media_id` 递归深度上限 10 层，超过时 UI 提示"已达最大嵌套深度"。

**说明：**
- 本地图的物理文件永远不删除，只删除数据库记录
- 生成图/截图进入回收站后永久删除时，同时删除物理文件
- 生成图可以多级嵌套（`parent_media_id` 指向另一张生成图）

### 3.5 Task（任务）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| workflow_type | enum | 工作流类型 |
| params | JSON | 工作流参数（含输入图路径） |
| status | enum | `pending` / `running` / `completed` / `failed` / `cancelled` |
| queue_order | int | 队列排序，支持拖拽调整 |
| created_at | datetime | 创建时间 |
| started_at | datetime? | 开始时间 |
| finished_at | datetime? | 完成/失败/取消时间 |
| error_message | string? | 失败原因 |
| result_media_ids | JSON? | 生成结果的 Media ID 列表 |
| execution_mode | enum | `immediate`（立即执行）/ `queued`（加入队列） |

### 3.6 QueueConfig（队列配置，全局单例）

| 字段 | 类型 | 说明 |
|------|------|------|
| start_mode | enum | `manual`（手动）/ `auto`（自动）/ `cron`（Cron 定时）/ `delay`（延时） |
| cron_expression | string? | Cron 表达式（仅 `cron` 模式生效，如 `0 23 * * *`） |
| delay_minutes | int? | 延时分钟数（仅 `delay` 模式生效） |
| is_paused | bool | 队列暂停标记（暂停后不执行新任务，已运行任务不受影响） |

**字段互斥规则：**
- `manual`：忽略 cron_expression 和 delay_minutes
- `auto`：忽略 cron_expression 和 delay_minutes
- `cron`：cron_expression 必填，忽略 delay_minutes
- `delay`：delay_minutes 必填，忽略 cron_expression

> 全局只有一条记录，控制整个任务队列的启动策略。

### 3.7 PoseAsset（动作资产）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| name | string | 动作名称 |
| tags | JSON | 标签列表（如["坐姿","户外","跟拍"]） |
| pose_image_path | string | DWPose 骨骼图路径 |
| created_at | datetime | 创建时间 |

**说明：**
- 用户上传原始参考图后，ComfyUI 提取 DWPose 骨骼图，仅保留骨骼图，原始图丢弃
- 骨骼图存入 `AppData/poses/` 目录
- 每个动作资产可被多个任务引用，任务记录中存储 `pose_asset_id`

### 3.8 PoseGroup（动作组）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| name | string | 动作组名称 |
| description | string? | 描述 |
| pose_ids | JSON | 有序的 PoseAsset ID 列表 |
| created_at | datetime | 创建时间 |

### 3.9 WorkspaceItem（工作区条目）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| media_id | UUID | 关联媒体 ID（外键指向 Media） |
| added_at | datetime | 加入时间 |
| sort_order | int | 排序（支持拖拽调整） |

**说明：**
- 持久化存储于 SQLite，刷新/关闭不会丢失
- 上限 100 条，超出时提示用户清理
- 删除条目只解除引用关系，不影响源媒体

### 3.10 PlatformAccount（平台账号）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| platform | enum | `xiaohongshu` / `bilibili` / `twitter` / `telegram` / `web` |
| username | string | 平台用户名（唯一标识） |
| display_name | string? | 显示名称 |
| person_id | UUID? | 关联人物（可为空，多账号可关联同一人物） |
| created_at | datetime | 首次创建时间 |

**说明：**
- 同一人物可关联多个平台账号（如小红书账号A + X账号B → 同一人物）
- 同一平台账号只能关联一个人物
- 账号与人物的关联关系在首次下载时询问，后续自动带入

### 3.11 DownloadRecord（下载记录）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| source_url | string | 原始链接（清洗后） |
| raw_text | string? | 原始粘贴文本（含分享提示文字） |
| platform | enum | 来源平台 |
| account_id | UUID? | 来源平台账号 |
| title | string? | 帖子/动态标题 |
| published_at | datetime? | 原帖发布日期 |
| media_count | int | 图片/视频数量 |
| album_id | UUID? | 生成的图集 ID |
| downloaded_at | datetime | 下载时间 |
| status | enum | `pending` / `completed` / `failed` |
| error_message | string? | 失败原因 |

### 3.12 数据库设计规范

**索引策略（SQLite）：**

| 表 | 索引 | 用途 |
|-----|------|------|
| Media | `(person_id, is_deleted)` | 人物主页查询 |
| Media | `(album_id, sort_order)` | 图集内排序展示 |
| Media | `(rating)` | 评分筛选/排序 |
| Media | `(parent_media_id)` | 生成链查询 |
| Media | `(is_deleted, deleted_at)` | 回收站列表及自动清理 |
| Task | `(status, queue_order)` | 队列调度 |
| WorkspaceItem | `(sort_order)` | 工作区展示 |
| DownloadRecord | `(status)` | 下载记录筛选 |

**并发策略：**
- SQLite 启用 WAL（Write-Ahead Logging）模式，支持多设备并发读取
- 写入操作由 FastAPI 后端统一管理，避免多客户端直接写入冲突
- 任务队列调度使用数据库乐观锁（`queue_order` + `status` 联合条件更新）

**Schema 迁移：**
- 使用 Alembic 管理数据库版本迁移
- 每次 schema 变更生成迁移脚本，启动时自动检测并执行
- `db/` 目录下保留 `alembic/versions/` 存放迁移历史

## 4. 目录结构

### 4.1 AppData 目录（可整体迁移，路径在设置中配置）

```
[AppData]/
  ├── db/
  │   └── main.sqlite
  ├── cache/
  │   └── thumbnails/          ← 缩略图缓存（可重建，不跟随迁移也无妨）
  ├── imports/
  │   └── clipboard/           ← 剪贴板导入图片的实际存储位置
  ├── generated/
  │   ├── upscale/             ← 高清放大结果
  │   ├── inpaint/             ← 局部修复/重绘结果
  │   ├── face_swap/           ← 换脸结果
  │   ├── portrait/            ← 写真生成结果
  │   └── screenshot/          ← 视频截图
  ├── poses/                   ← DWPose 骨骼图
  ├── workflows/               ← ComfyUI 工作流 JSON 模板
  └── downloads/               ← 网页抓取下载的图片/视频
      ├── xiaohongshu/
      ├── bilibili/
      ├── twitter/
      ├── telegram/
      └── web/
```

### 4.2 用户本地文件

任意位置的图片/视频文件夹，Motif 只存储绝对路径引用，不复制文件。

### 4.3 目录迁移

- 设置页提供"迁移 AppData 目录"功能
- 迁移时：复制整个 AppData 文件夹至新路径 → 更新 SQLite 中所有 `generated/` 和 `clipboard/` 的路径引用 → 更新设置中的 AppData 路径
- 用户本地图片的路径引用不受影响（文件未移动）

## 5. 各页面详细需求

### 5.1 左侧导航栏 `[P0]`

收起时显示图标，展开时显示图标 + 文字，默认收起。

```
[Motif Logo]
─────────────
📁 媒体库
📋 任务队列   ← 角标：左上旋转（运行中）/ 右上红色数字（失败）/ 右下绿色数字（完成，进入页面后清零）
🗂 工作区
🔧 小工具
🗑 回收站
⚙ 设置
─────────────
🔲 全屏切换   ← 桌面端侧边栏底部（ComfyUI 状态上方），调用 Fullscreen API
```

**移动端底部导航**：全屏按钮保留在 BottomNav 中，通过 `fullscreenchange` 事件监听器同步状态

### 5.2 媒体库 - 主页（人物一览） `[P0]`

**布局：** 网格卡片，支持 Ctrl+滚轮（桌面）或双指捏合（移动端）缩放网格列数。列数预设：`[20, 15, 12, 8, 5, 4, 3, 2, 1]`，默认 3 列，缩放后自动吸附最近预设值，持久化到 localStorage。

**人物卡片（PersonCard）：**
- 正方形封面（`aspect-square`），底部渐变叠加（`from-black/70`）显示白色人物名 + "N张·N图集"
- 左上角评分徽章（★4.2，半透明黑色背景），右上角 hover 显示 `…` 下拉菜单
- 紧凑模式（compact）：桌面端列数 ≥10 时、移动端列数 ≥4 时进入，隐藏渐变叠层文字和评分徽章
- 平均评分格式："★4.2 (12)"，括号内为已评分数量；全部未评分时不显示徽章

**排序方式（可切换）：**
- 按导入时间（默认）
- 按平均评分（未评分人物排末尾；同分时按导入时间降序）
- 按名称

**筛选：** 评分筛选（等于 / 大于等于 / 小于等于指定星级）、媒体类型（图片 / 视频，可叠加）

**顶部操作：**
- 新建人物
- 导入图片（弹出导入流程）
- 随机探索（全局范围，见 8.8 节）

**人物卡片右键菜单：**
- 重命名
- 设为封面（从该人物图片中选）

### 5.6 视频大图模式 `[P0]`

**布局：** 同图片大图模式，主区域替换为独立 `<VideoPlayer>` 组件（双 `<video>` 元素 A/B 交替，消除切换黑闪）

**播放器功能：**
- 播放/暂停、可点击/拖拽进度条（点击即跳转 + 拖拽 seek，含缓冲进度）、音量滑块、全屏、逐帧步进
- 倍速选择（0.5x–3x，PC 下拉菜单选择）
- 自动播放 + 静音策略：首次进入自动静音播放，用户手动取消静音后同会话内后续视频不再静音
- PC 键盘快捷键：空格（播放/暂停）、←/→（快退/快进 5s）、↑/↓（音量 ±10%）、F（全屏）
- **移动端控件栏精简**：采用溢出菜单布局，控件栏仅显示「播放/暂停 | 音量 | 时间 | ··· | 横屏」，次要功能（倍速、逐帧、截图）收入 `···` 弹出菜单。移动端横屏按钮等同全屏（不再单独显示全屏按钮）
- **右键菜单静音切换**：视频大图模式的右键菜单（移动端双指触碰）包含「取消静音/静音」选项，方便移动端切换

**移动端手势（Touch Arbiter 统一仲裁）：**

多种手势通过 `useTouchArbiter` 状态机统一仲裁，同一时刻仅允许一种手势生效：

```
idle → (touchstart) → pending
pending → (进度条 pointerdown) → seeking
pending → (横屏 + |dx|>5 && |dx|>|dy|) → seeking（拖拽 seek）
pending → (200ms 长按定时器，期间无水平移动) → speed_control
pending → (竖屏 + |dx|>5 && |dx|>|dy|) → swiping
pending → (touchend <300ms, 无移动) → tap → idle
seeking/swiping/speed_control → (touchend/pointerup) → idle
```

- **单击**（<300ms 无移动）：切换控件显示/隐藏
- **进度条拖拽**：pointerDown 立即 claim `seeking`，阻止其他手势
- **横屏水平拖拽视频区域**：claim `seeking`，拖拽距离占屏幕宽度比例映射到 min(duration, 600s) 进行 seek；拖拽开始时暂停视频避免反复缓冲，松开后恢复播放；拖拽期间强制显示进度条，松开后立即收起
- **手势优先级**：pending 状态下任何水平移动（>1px）立即取消长按定时器，确保拖拽 seek 优先于倍速触发
- **长按**（200ms，期间无移动）：进入 2x 倍速播放（claim `speed_control`）
- **长按中左右滑动**：调节倍速 0.5x–3x，步进 0.25x
- **长按后松开**：恢复原速 1x
- **长按后上滑再松开**：锁定当前倍速（顶部显示倍速 + 锁图标）
- **两指同时触碰**：弹出右键菜单（替代长按弹菜单，避免与倍速长按冲突；仅大图模式，其他页面仍用长按）
- **竖屏时左右滑动**（claim `swiping`）：由 LightBox 处理，切换上/下一张媒体
  - **速度阈值**：velocity > 0.3 px/ms 且 |dx| > 20px（快速轻扫）
  - **距离阈值**：|dx| > 60px（慢拖兜底）
  - 满足任一即触发切换

**横屏模式：**
- 移动端提供横屏切换按钮，先进入全屏再调用 `screen.orientation.lock('landscape')`，退出时 `unlock()` + `exitFullscreen()`
- 横屏时隐藏 TopBar（高度收为 0）+ 缩略图条，VideoPlayer 填满全屏
- 横屏时左右拖拽视频区域 = 拖拽 seek（不切换媒体）

**截图操作：** 控件栏提供截图按钮
- 截图存入 `generated/screenshot/`
- 截图作为子资产关联父视频（存储视频时间戳）
- 首张截图自动成为视频封面，多张截图时可手动指定封面
- 截图视为生成图，参与生成链管理
- **截图后可选高清放大**：截图完成后显示 Toast 通知"截图已保存"，附带"高清放大"操作按钮。用户可点击按钮打开 UpscaleDrawer 面板进行放大，也可忽略 Toast 保留原图。不强制要求放大。

**视频封面：** 默认第 1 帧，有截图后使用第一张截图，可手动指定

- 参数列表（键值对），媒体 ID 参数解析为缩略图预览 + 路径
- 错误信息（failed 时）
- 结果媒体列表（缩略图 + 路径）
- 通过 `POST /api/media/by-ids` 批量获取关联媒体信息

**最近完成结果视图：**
- 位于"已完成"区域上方，可折叠展开
- 收集最近 10 个已完成任务的 `result_media_ids`，批量获取 MediaItem
- 网格展示缩略图（6列），点击进入 LightBox 大图模式

**任务操作：**
- 拖拽调整队列顺序（仅对 `pending` 任务有效）：前端拖拽排序后调用 `PATCH /api/tasks/reorder` 批量更新 `queue_order`
- 重新启动（适用于 `failed` 任务）
- 删除任务

**任务失败处理：** 失败后自动跳过，继续执行下一个任务，任务标记为 `failed`，需手动重启

**添加任务时：** 弹出选择"立即执行"或"加入队列"

**导航角标：**
- 左上：运行中旋转动画
- 右上：失败数量（红色），归零需手动清除失败任务
- 右下：完成数量（绿色），进入任务队列页后清零
- 三种角标可同时重叠显示

## 6. 导入流程 `[P0]`

### 6.1 导入触发方式

- **粘贴导入：** 捕获全局剪贴板粘贴事件（`Ctrl+V`），导入单张图片，保存至 `AppData/imports/clipboard/`
- **文件选择（桌面端）：** 唤起系统文件选择器（tkinter），支持选择单张/多张媒体文件或文件夹。文件选择器默认显示"媒体文件"过滤器，包含图片和视频格式
- **文件上传（移动端）：** 移动端（触屏设备）使用 HTML `<input type="file" multiple accept="image/*,video/*">` 替代 tkinter 文件选择器，文件通过 HTTP multipart 上传到服务器，保存至 `AppData/imports/upload/`。移动端不支持子文件夹模式

### 6.2 导入逻辑

**选择单张或多张图片：**
1. 询问关联人物（链接到已有人物 / 新建人物 / 暂时无人物）
2. 询问是否新建图集或暂时无图集

**选择无子文件夹的文件夹：**
1. 将该文件夹视为一个图集导入
2. 询问是否更改图集名称（默认使用文件夹名）
3. 询问关联人物

**选择有子文件夹的文件夹（递归处理）：**
1. 展开导入确认弹窗，列出所有子文件夹（多层嵌套全部展平）
2. 每行显示：文件夹路径 → 图集名称（可编辑）→ 所属人物（可选）
3. 用户逐行确认或批量设置后提交

**文件夹名称智能拆分：**
- 当文件夹名包含下划线 `_`、横杠 `-` 或空格时，按**第一个**分隔符拆分
- 分隔符左侧作为默认**人物名**，右侧作为默认**图集名**
- 示例：`Alice_Wedding` → 人物"Alice"，图集"Wedding"；`Bob-旅行照片` → 人物"Bob"，图集"旅行照片"
- 若文件夹名不含上述分隔符，则人物名和图集名均默认为完整文件夹名（原有行为）
- 此规则适用于所有导入场景：无子文件夹的单文件夹、有子文件夹展平后的每一行、选择多张文件时的父文件夹名

**人物自动匹配与去重：**
- 拆分或推导出的人物名若与已有人物**名称匹配**（不区分大小写），自动选中该已有人物，而非新建
- 多个子文件夹拆分出相同人物名时（如 `Alice_生活照` 和 `Alice_写真集`），仅在第一次创建人物，后续自动复用，不会创建重复人物
- 用户仍可在导入弹窗中手动修改人物关联

### 6.3 导入进度与取消

- **进度显示：** 导入过程中显示进度条和计数（如"正在导入 45 / 230"）
- **可取消：** 导入过程中可随时取消，已导入的记录保留（不回滚）
- **大批量处理：** 超过 500 张图片时转为后台任务，顶部显示进度通知，用户可继续操作其他功能

### 6.4 人物上下文默认值

- 在**主页**触发导入：询问用户选择人物
- 在**人物主页**触发导入：默认关联当前人物，用户可强制更改

### 6.5 去重

- 以文件绝对路径作为唯一标识去重
- 重复扫描时跳过已存在记录，不重复导入

### 6.6 文件丢失处理 `[P1-后续]` 前端丢失标记待实现

- 系统检测到文件路径不可访问时，卡片显示丢失标记（红色虚线边框 + 警告图标）
- 支持两种重定向方式：
  - **单张重定向：** 右键菜单"重新定位文件"，选择新路径
  - **批量重定向：** 设置页或图集右键菜单，批量更新某图集/文件夹的路径前缀

### 6.7 导入弹窗设计原则

- 所有弹窗复用同一套设计组件（标题栏、操作区、确认/取消按钮）
- 弹窗层级最小化，避免嵌套弹窗
- 保持轻量，减少性能占用

## 8. 图片管理功能

### 8.1 评分系统

- 评分范围：1-5 星，可清除（清除后视为未评分）
- **平均分计算规则：仅计算已评分媒体的平均值，未评分媒体不参与计算**
- 全部未评分时，avg_rating 为 null，UI 显示"未评分"
- 前端显示格式：`★4.2 (12)` — 平均分 + 已评分数量
- 图集和人物的平均评分实时计算（基于下属已评分媒体）
- 评分入口：大图模式顶部、图片右键菜单、多选批量评分

### 8.2 筛选系统

筛选条件可叠加，在所有层级生效（包括大图模式的上下张切换）：

| 维度 | 选项 | 适用页面 |
|------|------|----------|
| 评分 | 全部 / 5星 / 4星+ / 3星+ / 2星以下 | 人物库、图集列表、散图、图集详情 |
| 来源类型 | 全部 / 本地图 / AI生成 / 截图 | 散图、图集详情 |

**筛选默认值配置：** 在设置页"筛选默认值"区域可配置评分筛选和来源类型的默认值（默认均为"全部"），存储在 localStorage `motif-filter-defaults`。每次进入页面时筛选条件自动重置为设置页配置的默认值。

### 8.3 排序系统

| 层级 | 排序选项 | 默认值 |
|------|----------|--------|
| 人物库 | 最新创建 / 评分最高 / 名称 A-Z | 最新创建 |
| 图集列表 | 最新创建 / 评分最高 / 名称 A-Z | 最新创建 |
| 散图 | 最新添加 / 评分最高 | 最新添加 |
| 图集详情 | 默认顺序 / 最新添加 / 评分最高 | 默认顺序 |

**评分排序规则：** 有评分的按平均评分降序排列，未评分（avg_rating = null）的排末尾；未评分组内按导入时间降序

**排序默认值配置：** 在设置页"筛选默认值"区域可为每个页面单独配置默认排序方式，存储在 localStorage `motif-filter-defaults`。每次进入页面时排序自动重置为设置页配置的默认值。

### 8.4 封面管理

- 人物封面：默认第一张图，右键图片"设为人物封面"
- 图集封面：默认第一张图，右键图片"设为图集封面"
- 视频封面：默认第 1 帧；有截图后使用第一张截图；多张截图时可手动指定

### 8.5 生成链管理 `[P1-后续]`

- 生成图支持多级嵌套（生成图的生成图）
- 任意生成图可脱离链接变为本地图（子数据：评分、二次生成图全部跟随）
- 脱离时物理文件不移动，数据库中解除关联关系，source_type 改为 `local`
- **脱离 API**：`POST /api/media/{id}/detach`，将 `parent_media_id`、`workflow_type`、`generation_params` 置为 null

### 8.6 软删除与回收站

- 删除操作均为软删除（标记 `is_deleted = true`，记录 `deleted_at`）
- 本地图：任何情况下只删除记录，物理文件不动
- 生成图/截图：软删除后进入回收站；回收站永久删除时同时删除物理文件
- 回收站支持恢复到原位置
- 回收站手动清空
- **自动清理：默认 30 天后自动永久删除，保留天数可在设置页配置（设为 0 关闭自动清理）**

### 8.7 批量清理低分生成图 `[P1-后续]`

**功能：** 批量删除当前作用域内评分低于指定值的生成图

**作用域：**
- 主页（全局）：清理所有人物下的低分生成图
- 人物主页：清理该人物下的低分生成图
- 图集详情页：清理该图集下的低分生成图

**触发入口：** 各层级右键空白处菜单

**清理规则：**
- 仅清理生成图（source_type 为 `generated` 或 `screenshot`），本地图不受影响
- **未评分的生成图永远不参与清理**
- 评分严格小于指定阈值的生成图将被删除

**操作流程：**
1. 弹出阈值设置面板（选择评分阈值，如"清理低于 3 星的生成图"）
2. 展示预览列表：将被删除的生成图缩略图及总数量
3. 用户二次确认后执行软删除（进回收站）

### 8.8 随机探索

**功能：** 在指定作用域和筛选条件内随机排列媒体，以大图模式浏览

**触发入口：** 各层级顶部工具栏"随机探索"按钮，自动继承当前作用域和已设筛选条件

**作用域：**
- 从触发层级自动继承（主页 = 全局，人物主页 = 该人物，图集页 = 该图集）
- 进入后可手动切换作用域（全局 / 指定人物 / 指定图集）

**筛选选项（可叠加）：**
- 评分：未评分 / 大于等于指定星级
- 媒体类型：图片 / 视频
- 来源类型：本地图 / 生成图

**探索行为：**
- 进入时对符合条件的媒体随机洗牌，生成本次探索序列
- 筛选条件变更后需手动点击"重新洗牌"（reshuffle），不动态应用
- 支持在序列内前进 / 后退切换
- 评分操作实时生效，但不重排本次序列

**界面：** 复用大图模式，顶部额外显示"探索进度"（如 12 / 87）和"重新洗牌"按钮

**沉浸模式：** 可在随机探索中开启沉浸模式（见 5.5.1 节）

## 9. 通用交互规范

### 9.1 右键菜单 vs 悬浮菜单

- PC：支持鼠标右键触发上下文菜单（拦截浏览器默认右键）
- 全平台：卡片右上角隐藏 `…` 按钮，悬浮 0.5s 后显示，点击触发同右键菜单
- 手机：长按触发多选，`…` 按钮触发菜单

### 9.2 多选交互

- PC：鼠标右键框选 / 点击复选框
- 手机：长按进入多选模式
- 多选支持的操作：移动到图集、批量加入工作区、批量删除、批量分配人物

### 9.3 参数面板（底部抽屉）

- 从屏幕底部滑入
- 手机端占满屏幕宽度，PC 端居中适当宽度
- 固定底部按钮：立即执行 / 加入队列

### 9.4 弹窗设计原则

- 复用同一套弹窗组件
- 避免嵌套弹窗
- 所有确认性操作（删除、脱离链接等）必须二次确认

### 9.5 缩略图

- 加载策略：懒加载（首次进入视口时生成）
- 尺寸：在设置中可配置（小 / 中 / 大）
- 缓存位置：`AppData/cache/thumbnails/`（可重建，迁移时不强制复制）
- 视频缩略图：后端 `/api/files/thumb` 使用 OpenCV 提取第一帧生成 JPEG 缩略图，走相同的缓存逻辑
- 视频封面优先级：`thumbnail_path`（截图封面）> 第一帧（实时提取）
- 视频卡片叠加播放三角图标（右下角），区分图片与视频。图标使用百分比尺寸（卡片宽度的 20%，最小 16px，最大 28px），随卡片大小自适应缩放

### 9.6 响应式与多平台

- PWA 支持，可添加到手机桌面
- 响应式布局断点：
  - `< 768px`：手机布局（单列，底部导航，全宽面板）
  - `768px - 1279px`：平板/小屏 PC（双列网格，侧边导航收起）
  - `1280px - 1919px`：标准 PC（多列网格，侧边导航可展开）
  - `≥ 1920px`：大屏 PC / 4K（更多列，更大缩略图）

**移动端页面头部/工具栏优化：**
- 所有页面头部（MediaLibrary、PersonHome、AlbumDetail）移动端使用图标按钮（文字通过 `hidden sm:inline` 隐藏）
- 移动端头部缩减内边距（`px-3`）、高度（`h-12`）和字号
- PersonHome hero 区域移动端：头像 64×64（桌面端 96×96），紧凑文字，操作按钮仅图标

**移动端内容区域：**
- 所有可滚动内容区域使用 `px-1 sm:px-6` 实现移动端最小侧边距
- 底部内边距 `pb-20 sm:pb-*` 为底部导航栏预留空间

**筛选栏（FilterBar）响应式：**
- 移动端：更小的下拉触发器（`w-24`/`w-20`，`h-7`，`text-[11px]`，`gap-1`）
- 桌面端：保持默认尺寸不变

**视频播放图标缩放：**
- MediaCard 视频播放图标使用百分比尺寸（卡片宽度的 20%，最小 16px，最大 28px），替代原固定 `w-7 h-7`，随卡片大小自适应

**紧凑模式阈值拆分：**
- 桌面端：列数 ≥10 时隐藏卡片信息（渐变叠层 + 评分徽章）
- 移动端：列数 ≥4 时隐藏卡片信息
- compact=true 时评分徽章同步隐藏（此前仅隐藏渐变文字）

- 手机端蒙版画笔支持触屏绘制（单指绘制 + 双指缩放平移），canvas 设置 `touch-action: none` 防止浏览器拦截触摸手势，使用 pointer capture 确保绘制连续性
- 手机端蒙版编辑器底部参数栏垂直堆叠布局，避免溢出；画笔大小默认 20（桌面端 40），范围 5-100（桌面端 5-200）
- 手机端导入使用 HTML file input 上传文件到服务器（`POST /api/media/upload-files`），无需 tkinter 文件选择器
- 手机端主要场景：浏览、评分筛选、任务管理、文件导入
- 手机端 AI 功能完整保留，但蒙版编辑精度有限
- 放大镜模式仅 PC 端支持，手机端使用双指缩放替代
- **网格缩放**：所有卡片网格页面支持双指捏合缩放（移动端）和 Ctrl+滚轮缩放（桌面端），每次 ±1 列连续调整，范围 1-30 列
- **缩放默认值**：设置页面可配置每个页面的默认列数，桌面端和手机端分别设置，支持任意正整数。每次切换页面时重置为默认值
- **防误触**：双指缩放结束后 300ms 内抑制点击事件，防止误触发卡片导航
- **手机端圆角**：手机端（`<640px`）去除所有卡片圆角
- **Gap 自动缩放**：桌面端按列数 1-2→16px / 3-5→8px / 6-8→6px / 9-12→4px / 13-15→2px / 16+→1px；手机端统一 1px
- **全屏模式**：桌面端在 Sidebar 底部（ComfyUI 状态上方）提供全屏按钮；移动端在 BottomNav 中提供全屏按钮，通过 `fullscreenchange` 事件监听器同步状态

## 10. 技术实现要点

### 10.1 工作流类型与命名规范

**workflow_type 枚举及文件名缩写：**

| 功能 | workflow_type | 文件名缩写 |
|------|--------------|-----------|
| 高清放大 | upscale | upscale |
| 局部修复 Flux（带提示词） | inpaint_flux | inpaint_f |
| 局部修复 SDXL（通用 A） | inpaint_sdxl | inpaint_s |
| 局部修复 Klein（通用 B） | inpaint_klein | inpaint_k |
| 换脸 | face_swap | faceswap |
| 图生图 | img2img | img2img |
| 写真生成 | portrait | portrait |
| DWPose 提取 | pose_extract | pose |

**文件命名规则（全局统一）：**
```
{参考图文件名}_{workflow_type缩写}_{序号}.png
示例：IMG_001_img2img_001.png
     IMG_001_faceswap_002.png
```

### 10.2 ComfyUI 集成

- 通过 `/prompt` 接口提交工作流 JSON
- 通过 `/history` 接口轮询任务状态（轮询间隔 2 秒）
- 工作流 JSON 模板存储在 `AppData/workflows/` 目录下，每个工作流类型一个 JSON 文件
- 参数占位符格式：`{{param_name}}`，如 `{{input_image_path}}`、`{{denoise_strength}}`
- 后端在提交前将占位符替换为实际参数值
- 任务严格串行，FastAPI 维护队列，一次只向 ComfyUI 提交一个任务
- 单任务超时阈值：默认 10 分钟，可在设置页配置

### 10.3 任务队列

- SQLite 存储任务记录和队列顺序
- FastAPI 后台线程维护队列调度
- 支持 4 种启动模式：手动（manual）/ 自动（auto）/ Cron 定时（cron，需 `croniter` 库解析表达式）/ 延时（delay）
- 支持队列暂停（`is_paused`），暂停后不执行新任务，已运行任务不受影响
- 延时模式从最后一个任务加入队列时刻起算
- 任务失败自动跳过，继续执行下一个
- 失败任务保留错误信息，支持手动重试

### 10.4 文件系统监控

- 导入时记录文件绝对路径，不复制文件
- 访问文件时检测路径可用性，不可用时显示丢失标记
- 剪贴板导入图片保存至 `AppData/imports/clipboard/`

### 10.5 放大镜实现

- 读取原始高分辨率图片，按指针位置裁剪对应区域渲染
- 放大层使用最高 z-index，覆盖所有 UI 元素
- 蒙版编辑器中禁用放大镜，改为画布整体缩放（滚轮缩放以指针为中心）

### 10.6 蒙版编辑器

- 使用 HTML5 Canvas 实现画笔绘制
- 蒙版以绿色半透明层叠加在图片上
- 支持撤销/重做（操作历史栈）
- 蒙版导出为 RGBA PNG（alpha=0 为修复区域，alpha=255 为保留区域），兼容 ComfyUI `LoadImage` 节点自动提取 alpha 通道作为 mask

### 10.7 视频处理

- HTML5 原生视频播放器，支持局域网串流
- 文件服务端点 `/api/files/serve` 必须支持 HTTP Range 请求（`206 Partial Content`），否则浏览器无法 seek 视频
- 截图使用 Canvas 从视频元素捕获当前帧
- 提供逐帧步进功能（前进/后退 1 帧）

### 10.8 生成链数据结构

- `parent_media_id` 自引用外键实现多级嵌套
- **最大递归深度 10 层**，后端在创建生成图时检查深度，达到上限时拒绝并提示"已达最大嵌套深度"
- 树状图渲染需递归查询子节点（后端提供 `/api/media/{id}/tree` 接口返回完整生成链）
- 脱离操作：更新 `source_type` 为 `local`、清空 `parent_media_id` 和 `workflow_type`

### 11.1 动作库 `[P2]`

#### 11.1.1 动作库主页

**未命名条目样式：** 混入列表但以特殊样式标记（灰色边框 + 警告图标），支持批量选中后统一打标签，名称各自单独填写

**筛选：** 按标签筛选，按名称搜索

**顶部操作：**
- 上传参考图（触发 DWPose 提取流程，见 11.1.2）
- 从媒体库选取（选择已入库的图片或图集生成骨骼图，见 11.1.2）
- 新建动作组

**动作卡片右键菜单：**
- 编辑名称/标签
- 加入动作组
- 查看使用此动作的全部生成图（见 11.1.3）
- 删除

#### 11.1.2 上传与提取流程

**来源方式一：上传本地文件**

- 单张图片：弹出命名+标签输入框，填写后提交任务（名称标签随任务传入，提取完成后直接入库）
- 文件夹：递归搜索文件夹内所有图片 → 展示预览确认列表（文件名 + 缩略图）→ 用户确认后批量加入队列

**来源方式二：从媒体库选取**

- 支持选择单张图片、多张图片或整个图集
- 选定后展示预览确认列表，确认后批量加入队列
- 单张时同样可以直接填写名称+标签随任务传入

**提取任务行为：**
- 每张图片作为独立任务加入公共串行任务队列（与其他 AI 任务混排，队列页显示任务类型为"DWPose 提取"）
- 提取失败的任务自动跳过，继续执行下一个
- 提取完成后仅保留骨骼图，原始参考图丢弃（媒体库来源的原图不受影响）
- 骨骼图存入 `AppData/poses/`
- 批量提取完成的条目在动作库显示为"待命名/待标签"状态

#### 11.1.3 动作使用记录视图

**触发：** 动作卡片右键菜单"查看使用此动作的全部生成图"

**展示：** 聚合展示所有使用该动作生成的图片（跨人物/跨图集），支持大图模式的全部功能（评分、AI 操作、生成链查看等）

#### 11.1.4 动作组管理

**功能：** 将多个动作资产编组，供写真生成批量使用

**动作组详情页：**
- 动作列表（有序，可拖拽排序）
- 添加/移除动作

### 11.2 网页图片抓取器 `[P3]`

**功能：** 解析社交平台帖子或含图网页链接，下载图片/视频并导入媒体库

**支持平台：**

| 平台 | 技术方案 | 授权方式 |
|------|----------|----------|
| 小红书 | 需登录态，图片有防盗链 | 账号 Cookie |
| B站动态/专栏 | API 较开放 | 账号 Cookie（部分内容需要） |
| X / Twitter | API 限制严格 | 账号 Cookie 或 API Key |
| Telegram | 公开频道直接抓取，私有频道需授权 | Telethon 账号授权 |
| 通用网页 | requests + Playwright 渲染动态页 | 无需授权 |

**各平台授权配置：** 统一在设置页"平台授权"分区配置 Cookie / API Key / Telethon 授权

#### 11.2.1 触发方式

**PC 端：**
- 在小工具页面的输入框粘贴链接，点击"解析"按钮

**手机端：**
- 同上，但粘贴内容可能包含分享提示文字（如"博主名发布了一条小红书笔记，快来看看吧~ http://xhslink.com/xxxxx"）
- 后端自动从粘贴文本中提取有效链接，支持多种平台的短链和完整链接格式

#### 11.2.2 下载流程

1. **粘贴文本 → 提取链接**
   - 后端从原始文本中正则提取有效 URL
   - 识别平台类型
   - 检查对应平台授权是否配置，未配置则提示前往设置

2. **解析元数据**（后台请求，前端显示加载状态）
   - 博主用户名 / 显示名
   - 帖子标题
   - 发布日期
   - 图片/视频预览（缩略图列表）
   - 媒体数量

3. **弹窗展示解析结果，用户确认**

4. **账号关联步骤**
   - 检查该平台账号是否已有关联人物记录
   - **已关联**：自动带入关联人物，用户可更改
   - **未关联**：询问关联到哪个人物（从已有人物选取 / 新建人物 / 暂不关联）
   - 提供"记住此账号的关联关系"选项（勾选后写入 PlatformAccount 表）

5. **图集配置**
   - 图集名称默认为帖子标题，可编辑
   - 选择归入方式：
     - 新建图集（默认）
     - 合并到已有图集（从该人物图集中选）
     - 作为散图（不归入图集）
   - 同一账号再次下载时：默认新建图集，可选已有图集

6. **确认下载**
   - 图片存入 `AppData/downloads/{platform}/`
   - 视频同样下载
   - 走正常导入流程，写入 Media 记录和 DownloadRecord 记录
   - 下载完成后跳转至生成的图集

#### 11.2.3 下载记录页

- 展示历史下载记录列表（来源平台、账号、标题、日期、状态）
- 失败记录显示错误原因，支持重试
- 点击记录可跳转至对应图集

#### 11.2.4 平台账号管理

**入口：** 设置页"平台账号"分区

**功能：**
- 查看所有已识别的平台账号列表
- 编辑账号与人物的关联关系
- 删除账号记录（不影响已下载的图片）

## 13. 异常处理规范

### 13.1 ComfyUI 异常

| 场景 | 预期行为 |
|------|----------|
| ComfyUI 进程未启动 | 前端显示"ComfyUI 启动中..."，AI 功能按钮禁用（"加入队列"仍可用），每 5 秒轮询连接状态 |
| ComfyUI 进程崩溃 | 后端检测到连接断开后标记状态为"连接失败"，当前运行中的任务标记为 `failed`（error_message = "ComfyUI 连接中断"），后端每 10 秒尝试重连，连续失败 30 次后停止重试并提示用户手动检查 |
| ComfyUI 任务超时 | 单任务超时阈值 10 分钟（可配置），超时后标记 `failed`，跳至下一任务 |

### 13.2 存储异常

| 场景 | 预期行为 |
|------|----------|
| 磁盘空间不足 | 生成任务执行前检查剩余空间（阈值 500MB），空间不足时拒绝执行并提示"磁盘空间不足，请清理后重试" |
| 文件路径不可访问 | 卡片显示丢失标记（红色虚线边框 + 警告图标），不阻塞其他功能 |
| 缩略图生成失败 | 显示默认占位图，不影响其他操作，后台重试一次 |

### 13.3 网络异常（局域网场景）

| 场景 | 预期行为 |
|------|----------|
| 客户端与后端断连 | 前端显示"连接中断，正在重连..."横幅，每 3 秒重试，恢复后自动刷新当前页面数据 |
| 请求超时 | 前端操作 10 秒无响应显示"操作超时"提示，用户可点击重试 |

### 13.4 外部平台异常（网页抓取器） `[P3]`

| 场景 | 预期行为 |
|------|----------|
| Cookie 过期 | 提示"授权已过期，请在设置页更新 Cookie" |
