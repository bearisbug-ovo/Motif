## 术语表

| 术语 | 含义 |
|------|------|
| 未分类 | 不属于任何图集的媒体，直接归属于某个人物或无人物（旧称"散图"） |
| 生成图 | 由 AI 工作流产出的图片（source_type = `generated`），区别于用户导入的本地图 |
| 生成链 | 一张本地图通过多次 AI 操作产生的树形派生关系（本地图 → 生成图 → 二次生成图…） |
| 脱离链接 | 将生成图从生成链中摘出，转变为独立的本地图，断开与来源图的派生关系 |
| 工作区 | 跨人物/跨图集的临时参考图收藏夹，持久化存储于 SQLite |
| 人脸参考图 | 换脸、写真等功能中提供面部特征的参考图片 |
| 风格参考图 | 换脸功能中提供姿态/场景/风格的目标图片 |

## 1. 项目概述

### 1.1 产品定位

Motif 是一款面向个人用户的本地图片/视频浏览、管理、AI 修复与联想生成工具。以 ComfyUI 作为 AI 能力后端，通过友好的 Web 界面提供照片整理、高清修复、换脸、写真生成等功能。

### 1.2 核心特性

- 以人物为核心的媒体管理体系
- 基于本地文件系统，不复制文件，只存储路径引用
- 通过 ComfyUI API 调用 AI 工作流，任务严格串行队列执行
- PWA + 响应式设计，支持 PC 与手机浏览器访问
- 支持多分辨率显示（1080p / 2K / 4K / 手机常用分辨率）
- 局域网多设备访问，FastAPI 绑定 `0.0.0.0`，无需认证

### 1.3 技术栈

| 层级 | 技术选型 |
|------|----------|
| 前端 | React + PWA，响应式布局 |
| 后端 | Python FastAPI（绑定 `0.0.0.0`，默认端口 `8000`） |
| 数据库 | SQLite（需配置 WAL 模式以支持多设备并发读取） |
| AI 后端 | ComfyUI（API 模式，`/prompt` 接口） |
| 启动方式 | 一键启动脚本（关闭旧进程 → 启动 FastAPI + ComfyUI → 前端轮询连接状态） |

## 2. 系统架构

### 2.1 整体架构

```
用户浏览器（PC / 手机）
      ↕ HTTP
FastAPI 后端
  ├── 文件系统管理（扫描、路径引用、缩略图）
  ├── SQLite 数据库（元数据 CRUD）
  ├── 任务队列管理（串行调度）
  └── ComfyUI 代理（/prompt API 调用）
      ↕ HTTP API
ComfyUI（本地运行）
  └── 各工作流节点
```

### 2.2 启动流程

用户双击 `start.bat`（或 `start.vbs`），启动图形化启动器（`launcher.py`，Python + tkinter），窗口打开后自动完成以下步骤：

1. **单实例检测**：通过命名 Mutex 防止重复启动，已有实例时弹窗提示并退出
2. **清理旧进程**：检测并关闭占用端口 8000 的旧进程
3. **构建前端**：执行 `npm run build`，生成生产包到 `frontend/dist/`（含 PWA Service Worker）。可在设置 tab 勾选"跳过前端构建"加速启动
4. **启动后端**：以隐藏窗口启动 `uvicorn`，日志写入 `.logs/backend.log`，等待端口 8000 就绪（超时 30 秒）。后端同时 serve 前端静态文件
5. **启动 ComfyUI**：检测 ComfyUI 是否已运行（请求 `/object_info/KSampler`），未运行则使用设置中的启动命令拉起。可在设置 tab 勾选"不启动 ComfyUI"跳过
6. **打开浏览器**：自动打开 `http://localhost:8000`

#### 启动器界面

启动器为 tkinter GUI 窗口（880×720，暗色主题，高 DPI 适配），包含三个 Tab：

**状态 Tab**：
- 三张状态卡片：后端服务（运行时间/PID/端口/错误统计）、ComfyUI（连接状态/地址）、网络（本机/局域网访问地址）
- 控制按钮：启动全部服务 / 停止全部 / 快速重启 / 打开浏览器
- 已连接设备列表：显示 30 分钟内活跃的客户端 IP、设备类型、请求数
- 状态每 5 秒自动刷新（通过后端 `/api/launcher/status`）

**设置 Tab**：
- 启动前可修改：后端端口、ComfyUI 地址和启动命令、缩略图大小
- 勾选项：跳过前端构建、不启动 ComfyUI

**日志 Tab**：
- 实时显示启动流程日志，带颜色标记（信息/成功/警告/错误）

#### 控制按钮

| 按钮 | 行为 |
|------|------|
| 启动全部服务 | 构建前端 + 启动后端 + 启动 ComfyUI + 打开浏览器（启动器打开时自动执行） |
| 停止全部 | 停止后端和 ComfyUI |
| 快速重启 | 重新构建前端 + 重启后端（不重启 ComfyUI，不打开浏览器） |
| 打开浏览器 | 打开 `http://localhost:{port}` |

#### 进程管理

- 启动器通过 `start.bat` → `start.vbs` → `pythonw.exe launcher.py` 启动，全程无 CMD 窗口
- 后端进程通过 Win32 Job Object 绑定到启动器，启动器退出时自动终止
- ComfyUI 作为独立进程启动，不受启动器生命周期影响
- 所有子进程使用 `CREATE_NO_WINDOW` 标志，不弹出控制台窗口
- 日志输出到项目根目录 `.logs/` 下：`backend.log`、`backend-error.log`

#### 前端 ComfyUI 状态

- 前端页面轮询 ComfyUI 连接状态，连接成功前 AI 功能按钮禁用，浏览功能不受影响
- 侧边栏底部显示 ComfyUI 连接状态指示器

#### 前端控制台（设置页 · 控制台 Tab）

设置页内嵌"控制台"Tab，提供 Web 端的服务监控与控制：
- 服务状态卡片（后端/ComfyUI/磁盘空间）+ 启动/停止/重启按钮
- 已连接设备列表（IP/UA/请求数/最后活跃时间）
- 错误统计（1h/24h 错误计数 + 最近 20 条错误详情）
- 后端日志查看

### 2.3 ComfyUI 连接状态

- **连接中**：旋转加载图标 + "ComfyUI 启动中..."
- **已连接**：绿色图标
- **连接失败 / 未连接**：红色图标 + 错误提示

**连接状态对功能的影响：**
- 浏览、管理、评分等功能不受影响
- AI 功能参数面板的"立即执行"按钮禁用，"加入队列"按钮保持可用
- 任务可以正常添加进队列，ComfyUI 恢复连接后按队列配置的启动模式继续执行

## 3. 数据模型

### 3.1 核心实体关系

```
人物 (Person) ──<>── 标签 (Tag)     ← 多对多（PersonTag）
  └──< 图集 (Album) ──<>── 标签 (Tag) ← 多对多（AlbumTag）
         └──< 媒体 (Media)  ←─────────────── 生成图也是 Media
                                              ↑
                                        关联人脸参考图
未分类（无图集的媒体，直属人物）
无人物未分类（导入时选择暂无人物）
```

### 3.2 Person（人物）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| name | string | 姓名 |
| cover_media_id | UUID? | 封面媒体 ID，默认第一张 |
| created_at | datetime | 创建时间 |
| avg_rating | float? | 加权平均评分（已评分用实际分，未评分按 2.5 计入；无媒体时为 null） |
| rated_count | int | 已评分媒体数量（用于前端显示"基于 N 张已评分"） |

### 3.3 Album（图集）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| person_id | UUID? | 所属人物，可为空（无人物图集） |
| name | string | 图集名称 |
| cover_media_id | UUID? | 封面，默认第一张 |
| created_at | datetime | 导入时间 |
| avg_rating | float? | 加权平均评分（已评分用实际分，未评分按 2.5 计入；无媒体时为 null） |
| rated_count | int | 已评分媒体数量 |
| is_generated_album | bool | 是否为 AI 生成图集（如批量 AI 任务结果、写真套图） |
| source_face_media_id | UUID? | 生成图集关联的人脸参考图 |

**生成图集显示**：图集详情页加载 is_generated_album=true 的图集时，自动清除 sourceType 筛选（生成图集内全部为 AI 生成图，若默认筛选为 local 则会显示为空）

### 3.4 Media（媒体）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| album_id | UUID? | 所属图集，可为空（未分类） |
| person_id | UUID? | 所属人物（**约束：当 album_id 不为空时，必须等于 Album.person_id**） |
| file_path | string | 文件绝对路径 |
| media_type | enum | `image` / `video` |
| source_type | enum | `local`（本地图）/ `generated`（生成图）/ `screenshot`（视频截图） |
| parent_media_id | UUID? | 生成图/截图关联的来源媒体 ID（截图时指向父视频，通过父视频的 media_type=video 区分） |
| video_timestamp | float? | 截图对应视频时间戳（秒），仅 source_type=screenshot 时有值 |
| workflow_type | enum? | 使用的工作流类型 |
| generation_params | JSON? | 生成参数快照 |
| rating | int? | 评分 1-5，null 表示未评分 |
| sort_order | int | 图集内排序 |
| playback_position | float? | 视频播放进度（秒），用于跨设备续播 |
| imported_at | datetime | 导入/生成时间 |
| is_deleted | bool | 软删除标记 |
| deleted_at | datetime? | 删除时间 |
| thumbnail_path | string? | 缩略图缓存路径（可重建） |

**数据完整性约束：**
- `person_id` 一致性：当 Media 属于某 Album 时，Media.person_id 必须与 Album.person_id 相同。后端在导入和移动操作时强制校验。
- 生成链深度限制：`parent_media_id` 递归深度上限 10 层，超过时 UI 提示"已达最大嵌套深度"。

**换脸结果归属规则**：
- person_id 优先级：face_ref.person_id > target_person_id > source.person_id（换脸结果归属于人脸参考图的人物）
- album_id 仅在与 person_id 属于同一人物时继承，否则设为 null（散图）
- 批量换脸时自动为 **人脸参考图所属人物** 创建生成图集（is_generated_album=True），名称格式 "换脸 - {源图集名}"
- 链式任务（复合工作流）中 target_person_id 和 result_album_id 传播到所有步骤

**说明：**
- 本地图的物理文件永远不删除，只删除数据库记录
- 生成图/截图进入回收站后永久删除时，同时删除物理文件
- 生成图可以多级嵌套（`parent_media_id` 指向另一张生成图）

### 3.5 Task（任务）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| workflow_type | enum | 工作流类型 |
| params | JSON | 工作流参数（含输入图路径） |
| status | enum | `pending` / `running` / `completed` / `failed` / `cancelled` |
| queue_order | int | 队列排序，支持拖拽调整 |
| created_at | datetime | 创建时间 |
| started_at | datetime? | 开始时间 |
| finished_at | datetime? | 完成/失败/取消时间 |
| error_message | string? | 失败原因 |
| result_media_ids | JSON? | 生成结果的 Media ID 列表 |
| result_outputs | JSON? | 输出结果，包含文本输出（如反推提示词）和图片输出（`{"type": "image", "path": "..."}`），格式 `{"key": value}` |
| execution_mode | enum | `immediate`（立即执行）/ `queued`（加入队列） |
| chain_id | UUID? | 链式任务组 ID，同一链的所有步骤共享此 ID |
| chain_order | int? | 链内步骤序号（0 = 第一步，1 = 第二步…） |
| chain_source_param | string? | 接收上一步结果的参数名（如 `source_media_id`），第一步为 null |

### 3.6 QueueConfig（队列配置，全局单例）

| 字段 | 类型 | 说明 |
|------|------|------|
| start_mode | enum | `manual`（手动）/ `auto`（自动）/ `cron`（Cron 定时）/ `delay`（延时） |
| cron_expression | string? | Cron 表达式（仅 `cron` 模式生效，如 `0 23 * * *`） |
| delay_minutes | int? | 延时分钟数（仅 `delay` 模式生效） |
| is_paused | bool | 队列暂停标记（暂停后不执行新任务，已运行任务不受影响） |

**字段互斥规则：**
- `manual`：忽略 cron_expression 和 delay_minutes
- `auto`：忽略 cron_expression 和 delay_minutes
- `cron`：cron_expression 必填，忽略 delay_minutes
- `delay`：delay_minutes 必填，忽略 cron_expression

**delay 模式行为**：
- 收到新任务后开始 debounce 计时（默认 5 分钟）
- 计时期间如有新任务加入，重置计时器
- 计时期间如收到手动启动信号（立即执行），立即开始处理
- 使用 1 秒轮询循环实现，不阻塞队列运行器

> 全局只有一条记录，控制整个任务队列的启动策略。

### 3.7 PoseAsset（动作资产）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| name | string | 动作名称 |

| pose_image_path | string | DWPose 骨骼图路径 |
| created_at | datetime | 创建时间 |

**说明：**
- 用户上传原始参考图后，ComfyUI 提取 DWPose 骨骼图，仅保留骨骼图，原始图丢弃
- 骨骼图存入 `AppData/poses/` 目录
- 每个动作资产可被多个任务引用，任务记录中存储 `pose_asset_id`

### 3.10 PlatformAccount（平台账号）

| 字段 | 类型 | 说明 |

| username | string | 平台用户名（唯一标识） |
| display_name | string? | 显示名称 |

### 3.13 Tag（标签）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| name | string | 标签名称（唯一） |
| color | string? | 标签颜色（HEX，如 `#FF6B6B`），为空时使用默认色 |
| sort_order | int | 排序权重（越小越靠前） |
| created_at | datetime | 创建时间 |

### 3.14 PersonTag（人物-标签关联）

| 字段 | 类型 | 说明 |
|------|------|------|
| person_id | UUID | 外键 → Person.id，联合主键 |
| tag_id | UUID | 外键 → Tag.id，联合主键 |

### 3.15 AlbumTag（图集-标签关联）

| 字段 | 类型 | 说明 |
|------|------|------|
| album_id | UUID | 外键 → Album.id，联合主键 |
| tag_id | UUID | 外键 → Tag.id，联合主键 |

**说明：**
- 人物和图集均支持多标签，多对多关系
- 删除标签时级联删除所有关联记录（PersonTag / AlbumTag）
- 合并标签时将源标签的所有关联转移到目标标签，去重后删除源标签

### 3.16 数据库设计规范

**索引策略（SQLite）：**

| 表 | 索引 | 用途 |
|-----|------|------|
| Media | `(person_id, is_deleted)` | 人物主页查询 |
| Media | `(album_id, sort_order)` | 图集内排序展示 |
| Media | `(rating)` | 评分筛选/排序 |
| Media | `(parent_media_id)` | 生成链查询 |
| Media | `(is_deleted, deleted_at)` | 回收站列表及自动清理 |
| Task | `(status, queue_order)` | 队列调度 |
| WorkspaceItem | `(sort_order)` | 工作区展示 |
| DownloadRecord | `(status)` | 下载记录筛选 |
| PersonTag | `(person_id)` | 人物标签查询 |
| PersonTag | `(tag_id)` | 按标签筛选人物 |
| AlbumTag | `(album_id)` | 图集标签查询 |
| AlbumTag | `(tag_id)` | 按标签筛选图集 |

**并发策略：**
- SQLite 启用 WAL（Write-Ahead Logging）模式，支持多设备并发读取
- 写入操作由 FastAPI 后端统一管理，避免多客户端直接写入冲突
- 任务队列调度使用数据库乐观锁（`queue_order` + `status` 联合条件更新）

**Schema 迁移：**
- 使用 Alembic 管理数据库版本迁移
- 每次 schema 变更生成迁移脚本，启动时自动检测并执行
- `db/` 目录下保留 `alembic/versions/` 存放迁移历史

### 5.5 大图浏览模式 `[P0]`

**触发：** 点击任意图片卡片

**整体布局：**

```
┌─────────────────────────────────────────────────┐
│  [返回]  [评分★]  [AI功能按钮组]  [移动][封面][工作区][文件管理器][删除]  │ ← 顶部操作栏
├─────────────────────────────────────────────────┤
│                                                 │
│                  主图展示区                       │
│             （放大交互区域）                       │
│                                                 │
├─────────────────────────────────────────────────┤
│     生成链指示器（常驻，有生成图时显示）              │
├─────────────────────────────────────────────────┤
│     底部预览条（仅本地图，动态滚动高亮）              │
└─────────────────────────────────────────────────┘
```

**顶部操作栏：** 与图片右键菜单保持功能一致，布局更合理

**底部预览条：**
- 始终只显示本地图（这是布局设计，不受来源类型筛选控制），评分和媒体类型筛选生效
- 无图集/人物上下文时（如从任务队列、工作区打开），显示传入的全部 items
- 动态滚动，当前本地原图高亮居中（若当前图是生成图，高亮其所属的本地原图）
- 点击缩略图 = 水平轴导航，直接跳转到该本地图

**双轴导航模型：** 大图模式提供两种独立的切换方式——**本地图导航（水平轴）** 和 **生成链导航（垂直轴）**，所有筛选中评分和媒体类型筛选生效，来源类型筛选不影响切换（以保证生成链遍历完整）。

**水平轴 — 本地图导航：** 在本地图序列中切换，跳过所有生成图。

| 优先级 | 当前位置 | 下一张 | 终止条件 |
|--------|---------|--------|----------|
| 1 | 任意图（含生成图） | 当前图所属本地原图的同图集内下一张本地图（按 sort_order） | 图集末尾时跳至优先级 2 |
| 2 | 图集末尾 | 同人物下一个图集的第一张本地图 | 人物下无更多图集时跳至优先级 3 |
| 3 | 人物末尾 | 下一个人物的第一张本地图 | 所有人物遍历完毕时提示"已到最后一张" |

> 若当前图是生成图，先沿 parent_media_id 回溯到本地原图，再从该本地图位置执行水平导航。

**无上下文降级（扁平模式）：** 从任务队列、工作区等无图集/人物上下文的入口打开大图时，水平轴退化为在传入的 items 扁平数组中顺序切换（所有 items 视为同一概念图集），不执行跨图集/跨人物跳转。垂直轴（生成链）仍正常工作。随机探索模式同样使用扁平模式（保持随机排列顺序）。

**垂直轴 — 生成链导航：** 在当前本地图的生成链中深度优先遍历。

| 优先级 | 当前位置 | 下一张（↓） | 终止条件 |
|--------|---------|-------------|----------|
| 1 | 任意图 | 当前图的第一张直接子生成图 | 无子生成图时跳至优先级 2 |
| 2 | 生成图 | 同父图的下一张兄弟生成图 | 无更多兄弟时跳至优先级 3 |
| 3 | 无更多兄弟 | 逻辑回溯到父图，继续找父图的下一个兄弟（递归向上） | 回到本地原图（根节点）且无更多子节点时跳至优先级 4 |
| 4 | 生成链遍历完毕 | 自动切换到下一张本地图（等同水平轴 →） | 与水平轴终止条件一致 |

> **回溯说明**：优先级 3 是逻辑回溯，不切换显示。例如生成链 `原图A → 放大1 → 换脸1`，在换脸1 按 ↓，无子代 → 无兄弟 → 回溯到放大1 → 找放大1 的下一个兄弟"放大2" → 切换到放大2。若放大1 也无更多兄弟 → 回溯到原图A → 找原图A 的下一个子代"放大3"（若有） → 否则生成链遍历完毕 → 切换下一张本地图。

上一张（↑）反向遍历，规则对称。在本地原图（根节点）按 ↑ 切换到上一张本地图（等同水平轴 ←）。无生成图的本地图按 ↓ 直接切换下一张本地图。

> 生成链递归最大深度 10 层，超过时跳过深层子节点。

**生成链指示器（常驻，替代原右侧面板）：** 主图区与底部预览条之间，常驻显示当前图在生成链中的位置。

*默认态（收起）：*
- 紧凑单行横向排列：`[本地原图] → [子1] [子2] ... → [孙1] ...`，缩略图小圆角（24×24），用连线和缩进表达层级
- 当前图高亮边框（primary 色），其余半透明
- 不同工作流类型用不同边框颜色区分（upscale=蓝, face_swap=紫, inpaint=琥珀, screenshot=绿）
- 无生成图时整行不显示（高度收为 0）
- 点击任意节点 = 跳转到该图（垂直轴导航）
- 滚轮在该区域上滚动 = 垂直轴生成链导航

*展开态（点击展开按钮）：*
- 展开为浮层水平树布局（floating overlay），横向 = 生成深度（列），纵向 = 同一父节点的子节点按创建时间从上到下排列
- 节点以缩略图（40×40）展示，底部叠加工作流类型标签，SVG 贝塞尔曲线连接父子节点
- 点击外部区域收起（click-outside-to-close）
- 支持在节点上右键"脱离生成链"
- 生成链递归最大深度 10 层，超过时折叠显示

*功能覆盖（原右侧面板全部功能）：*
- ✅ 可视化展示完整生成链树状层级（展开态水平树）
- ✅ 不同颜色区分工作流类型
- ✅ 当前图高亮 + 点击跳转
- ✅ 脱离生成链操作（展开态右键菜单）

**轴切换视觉反馈：** 不使用滑入/滑出动画（性能问题），采用轻量方案：
- 水平轴切换：底部预览条高亮闪烁一次（`opacity 0.6 → 1`，150ms），提示用户在本地图序列中移动
- 垂直轴切换：生成链指示器高亮闪烁一次，同时当前节点边框 pulse 动画（150ms）
- 图片本身无方向性过渡动画，直接替换（crossfade 也不加，避免累积延迟）

**键盘快捷键：**
- `←` / `→`：本地图导航（水平轴）；焦点在 input/textarea 内时不拦截
- `↑` / `↓`：生成链导航（垂直轴）；焦点在 input/textarea 内时不拦截
- `Esc`：退出大图模式 / 退出沉浸模式
- `1`-`5`：快速评分
- `0`：清除评分
- 放大模式下按方向键：导航图片但保持缩放状态（不再先退出放大模式）

**鼠标操作：**
- 单击主图：进入放大模式（以点击位置为中心放大到 2x，带动画）
- 单击黑边区域：退出大图模式（不触发放大）
- 单击（已放大时）：退出放大模式（动画回到 fit）
- 双击主图：切换 fit ↔ 2x（带动画）
- 滚轮（图片上）：生成链导航（垂直轴）；若无生成链则以鼠标为中心缩放（1x ~ 8x）
- 滚轮（黑边区域）：生成链导航（垂直轴）
- 滚轮（底部预览条）：横向滚动预览条本身（不触发导航）
- 滚轮（生成链指示器）：生成链导航（垂直轴）
- 滚轮（弹窗覆盖层）：不触发导航（`[role="dialog"]` 内的滚轮事件被屏蔽，如 MediaDetailDialog）
- 鼠标移动（已放大时）：图片跟随鼠标平移，鼠标在原图上的位置点与放大图对应位置重合；鼠标指针显示为放大镜图标；鼠标移出原图范围时隐藏放大图并恢复默认指针
- 右键（已放大时）：退出放大模式

**左右切换导航条（PC）：**
- 左右边缘各显示一个全高导航长条（`w-28`），hover 时显示半透明背景（`bg-white/20`）和方向箭头图标
- 鼠标指针替换为 SVG 方向箭头（CSS `cursor: url(data-uri)` 原生渲染）
- 到达首张/末张时对应方向导航条不显示
- 仅 PC 端显示，移动端使用滑动手势切换
- 导航条点击 = 本地图导航（水平轴）

#### 5.5.1 沉浸模式（全屏浏览）

**触发：** 顶部操作栏沉浸模式按钮，普通大图模式和随机探索均可开启

**开启后：** 隐藏所有 UI（顶部栏、底部预览条、生成链指示器、左侧导航），仅保留图片全屏展示

**保留操作：**
- 本地图导航：键盘 `←` / `→`；手机左右滑动
- 生成链导航：键盘 `↑` / `↓`；手机上下滑动
- 评分：键盘 `1`-`5`；手机**双击**图片弹出评分浮层（点击星级后浮层消失）

**退出沉浸模式：**
- PC：鼠标右键弹出菜单 → 选择"退出沉浸模式"；或按 `Esc`
- 手机：两指同时触碰弹出菜单 → 选择"退出沉浸模式"（大图模式统一使用两指触碰替代长按弹菜单，避免与视频倍速手势冲突）

#### 5.5.2 放大模式

**PC 端（鼠标）：**
- **触发：** 左键单击主图区 / 滚轮向上 / 双击
- **行为：** 不再使用覆盖层，直接对主图 CSS transform 缩放，以触发点为中心放大
- **缩放：** 滚轮在图片上以鼠标位置为中心缩放（1x ~ 8x），zoom factor = 1.15/tick
- **平移：** 已放大状态下鼠标移动即平移图片，鼠标在原图上的位置点与放大图对应位置精确重合；鼠标指针替换为 SVG 放大镜图标（CSS `cursor: url(...)` 原生渲染）；鼠标移出原图范围时隐藏放大图并恢复默认指针；无边界约束
- **双击：** 切换 fit ↔ 2x（带 0.3s 缓动动画）
- **退出：** 单击 / 右键 / Esc 退出放大模式（动画回到 fit-to-screen）
- **方向键（放大中）：** ← → ↑ ↓ 在放大模式下直接执行导航（水平/垂直轴），保持当前缩放比例不变
- **滚轮区域分离：** 黑边和缩略条区域滚轮切换图片，图片区域滚轮缩放
- **性能：** CSS transform + will-change + rAF 直接 DOM 操作，60fps

**移动端（触摸）：**
- **触发：** 双指捏合（pinch-to-zoom）放大图片
- **行为：** 图片以捏合中心为原点缩放（1x ~ 8x），支持双指平移
- **单指拖拽：** 放大状态下单指可拖拽平移浏览
- **退出：** 单击图片恢复原始大小；缩放接近 1x 时自动 snap 回原始尺寸
- **仅图片：** 视频不支持放大模式

**禁用时机：** 蒙版编辑器中禁用放大模式，改为画布整体缩放

### 5.6 视频大图模式 `[P0]`

**布局：** 同图片大图模式，主区域替换为独立 `<VideoPlayer>` 组件（双 `<video>` 元素 A/B 交替，消除切换黑闪）

**关闭行为：** 与图片大图一致 —— 点击视频画面内的 letterbox 黑边区域可关闭大图模式，点击视频内容区域不关闭

**播放器功能：**
- 播放/暂停、可点击/拖拽进度条（点击即跳转 + 拖拽 seek，含缓冲进度）、音量滑块、全屏、逐帧步进
- 倍速选择（0.5x–3x，PC 下拉菜单选择）
- 自动播放 + 静音策略：首次进入自动静音播放，用户手动取消静音后同会话内后续视频不再静音
- PC 键盘快捷键：空格（播放/暂停）、←/→（快退/快进 5s）、F（全屏）
- **视频模式键盘轴分配：** ← → 归 VideoPlayer（快退/快进），不触发水平轴导航（水平轴通过底部预览条/导航条/移动端滑动操作）；↑ ↓ 竖屏时用于生成链导航（如视频截图属于视频的生成子代），横屏时归 VideoPlayer（音量 ±10%）
- **移动端控件栏精简**：采用溢出菜单布局，控件栏仅显示「播放/暂停 | 音量 | 时间 | ··· | 横屏」，次要功能（倍速、逐帧、截图）收入 `···` 弹出菜单。移动端横屏按钮等同全屏（不再单独显示全屏按钮）
- **右键菜单静音切换**：视频大图模式的右键菜单（移动端双指触碰）包含「取消静音/静音」选项，方便移动端切换

**移动端手势（Touch Arbiter 统一仲裁）：**

多种手势通过 `useTouchArbiter` 状态机统一仲裁，同一时刻仅允许一种手势生效：

```
idle → (touchstart) → pending
pending → (进度条 pointerdown) → seeking
pending → (横屏 + |dx|>5 && |dx|>|dy|) → seeking（拖拽 seek）
pending → (200ms 长按定时器，期间无水平移动) → speed_control
pending → (竖屏 + |dx|>5 && |dx|>|dy|) → swiping
pending → (touchend <300ms, 无移动) → tap → idle
seeking/swiping/speed_control → (touchend/pointerup) → idle
```

- **单击**（<300ms 无移动）：切换控件显示/隐藏
- **进度条拖拽**：pointerDown 立即 claim `seeking`，阻止其他手势
- **横屏水平拖拽视频区域**：claim `seeking`，拖拽距离占屏幕宽度比例映射到 min(duration, 600s) 进行 seek；拖拽开始时暂停视频避免反复缓冲，松开后恢复播放；拖拽期间强制显示进度条，松开后立即收起
- **手势优先级**：pending 状态下任何水平移动（>1px）立即取消长按定时器，确保拖拽 seek 优先于倍速触发
- **长按**（200ms，期间无移动）：进入 2x 倍速播放（claim `speed_control`）
- **长按中左右滑动**：调节倍速 0.5x–3x，步进 0.25x
- **长按后松开**：恢复原速 1x
- **长按后上滑再松开**：锁定当前倍速（顶部显示倍速 + 锁图标）
- **两指同时触碰**：弹出右键菜单（替代长按弹菜单，避免与倍速长按冲突；仅大图模式，其他页面仍用长按）
- **竖屏时左右滑动**（claim `swiping`）：由 LightBox 处理，本地图导航（水平轴）
  - **速度阈值**：velocity > 0.3 px/ms 且 |dx| > 20px（快速轻扫）
  - **距离阈值**：|dx| > 60px（慢拖兜底）
  - 满足任一即触发切换
- **竖屏时上下滑动**（claim `swiping_vertical`）：由 LightBox 处理，生成链导航（垂直轴）
  - 阈值同左右滑动（方向取 |dy|）
  - 下滑 = 生成链下一张（深入子代），上滑 = 生成链上一张（回到父图）

**横屏模式：**
- 移动端提供横屏切换按钮，先进入全屏再调用 `screen.orientation.lock('landscape')`，退出时 `unlock()` + `exitFullscreen()`
- 横屏时隐藏 TopBar（高度收为 0）+ 缩略图条，VideoPlayer 填满全屏
- 横屏时左右拖拽视频区域 = 拖拽 seek（不切换媒体）

**截图操作：** 控件栏提供截图按钮
- 截图存入 `generated/screenshot/`
- 截图作为子资产关联父视频（存储视频时间戳）
- 首张截图自动成为视频封面，多张截图时可手动指定封面
- 截图视为生成图，参与生成链管理
- **截图后可选高清放大**：截图完成后显示 Toast 通知"截图已保存"，附带"高清放大"操作按钮。用户可点击按钮打开 WorkflowRunDialog（category=upscale）进行放大，也可忽略 Toast 保留原图。不强制要求放大。

**播放进度记忆（跨设备）：**
- 播放位置存储在数据库（Media.playback_position），多设备共享同一进度
- **自动保存时机**：暂停时、每 15 秒定时、离开视频时（切换/关闭大图）
- **自动恢复**：打开视频后自动跳转到上次位置，顶部显示"从 X:XX 继续播放"提示（2.5 秒后消失）
- **播完清除**：播放至距结尾 3 秒内时自动清除进度，下次从头播放
- **防抖写入**：位置变化 > 2 秒才触发保存，避免频繁写库

**视频封面：** 默认第 1 帧，有截图后使用第一张截图，可手动指定

### 5.8 任务队列页 `[P1]`

**布局：**

*桌面端（md+）：* 双栏布局
- 左栏：页头（暂停/恢复按钮 + 清空未完成按钮 + 队列启动模式配置） + 任务列表（按状态分组）
- 右栏：最近完成结果（sticky 侧边栏，3 列网格，默认折叠 6 行，可展开）

*移动端：* 单栏布局
- 页头：按钮为图标模式（32px 正方形，无文字），sm+ 显示完整文字。内容区内边距响应式
- 任务列表中，最近完成结果内嵌于"失败"和"已完成"区域之间（4 列网格，默认折叠 2 行，可展开/收起）

*排序：* 已完成/失败任务按 `finished_at` 降序排列（最新完成的排在最前）

**页头按钮：**
- **暂停/恢复队列**：调用 `PUT /api/queue/config` 设置 `is_paused` 字段。暂停时按钮显示为琥珀色（amber），队列暂停后不会自动取下一个任务执行
- **清空未完成**（"清空未完成"）：一键删除所有 pending + failed + cancelled 状态的任务，调用 `POST /api/tasks/bulk-delete`，返回删除数量

**队列启动模式（4 种，与 QueueConfig.start_mode 一一对应）：**
- **手动启动**（`manual`）：点击"开始执行"按钮触发
- **自动启动**（`auto`）：有待执行任务即自动开始
- **Cron 定时**（`cron`）：配置 Cron 表达式（如 `0 23 * * *` 表示每天 23:00）
- **延时启动**（`delay`）：配置 X 分钟后启动（从最后一个任务加入队列时刻起算）

**任务卡片显示：**
- 工作流类型、输入图预览、关键参数摘要
- 状态：待执行 / 执行中（实时进度条 + value/max 文字）/ 已完成 / 失败（显示错误原因）
- 执行时间
- 进度信息：通过 `GET /api/tasks/stats` 返回的 `progress` 字段（`{task_id, value, max}`）驱动，3秒轮询更新

**右键菜单（按状态动态显示）：**
- **所有状态**：查看详情（弹出任务详情对话框）
- **所有非 running 状态**（仅 custom 工作流）：编辑参数并新建（打开 WorkflowRunDialog，预填该任务的工作流和参数，用户编辑后作为新任务提交）
- **pending**：取消、删除
- **running**：取消
- **completed**：重新执行、查看结果（在 LightBox 中打开结果图）、删除
- **failed/cancelled**：重试（创建新任务，保留原任务历史）、删除

**任务详情对话框：**
- 显示任务类型、状态、创建/开始/完成时间、耗时
- 参数列表（键值对），媒体 ID 参数解析为缩略图预览 + 路径
- 错误信息（failed 时）
- 输出参数（`result_outputs`，非图片输出如反推提示词等文本/数值结果）
- 结果媒体列表（缩略图 + 路径）
- 通过 `POST /api/media/by-ids` 批量获取关联媒体信息

**最近完成结果视图（RecentResultsGrid 组件）：**
- 收集最近 50 个已完成任务的 `result_media_ids`，按 `finished_at` 降序排列，批量获取 MediaItem（`getByIds` 结果按请求顺序重排）
- 桌面端：右栏 sticky 侧边栏，3 列网格，默认折叠显示 6 行；侧边栏高度限制为视口高度，展开后超出部分可滚动
- 移动端：内嵌于"失败"和"已完成"任务区之间，4 列网格，默认折叠 2 行，支持展开/收起
- 点击进入 LightBox 大图模式（`taskResultsMode`：禁用生成链 UI，改为源图导航按钮）
- 右键菜单与图片卡片一致（AI 功能、加入工作区、移动到图集、资源管理器、评分、查看详情、删除），但隐藏"生成链"和"脱离生成链"菜单项
- **源图导航按钮**（替代生成链指示器，浮于媒体区底部中央）：
  - 「查看源图 / 返回结果」：切换显示当前结果图的源图（`parent_media_id`），查看源图时按钮高亮蓝色，左右导航时自动退出源图查看
  - 「在图集中查看」：关闭当前 LightBox，以源图所在的图集/人物上下文重新打开（支持完整双轴导航）

**批量任务分组（Batch Grouping）：**
- 通过 `POST /api/tasks/batch` 创建的任务共享同一 `batch_id`（UUID）
- 在待执行（pending）区域，同一 `batch_id` 的任务折叠为一张分组卡片，显示 Layers 图标 + 任务数量
- 点击分组卡片展开显示组内各个任务，再次点击收起
- 非 pending 状态的任务不分组，正常显示

**运行中区域折叠：**
- 当运行中的任务超过 2 个时，运行中区域可折叠，折叠后仅显示 1 个任务 + 剩余数量提示

**任务操作：**
- 重新启动（适用于 `failed`/`cancelled`/`completed` 任务，创建新任务而非重置原任务，保留历史记录）
- 删除任务
- 批量删除未完成任务（页头"清空未完成"按钮）

**任务失败处理：** 失败后自动跳过，继续执行下一个任务，任务标记为 `failed`，需手动重启

**链式任务（Task Chaining）：** 允许用户将两个工作流步骤串联执行（如图生图 → 高清放大），原子性地按顺序完成：
- **创建方式：** WorkflowRunDialog 中点击"链接下一步"按钮，选择第二步的工作流类别、具体工作流及参数
- **执行机制：** 同一 `chain_id` 的步骤按 `chain_order` 顺序原子执行，不允许其他任务插入中间；第二步通过 `chain_source_param` 自动接收第一步的输出结果
- **成功处理：** 所有步骤都成功时，最终结果 reparent 到原始源图（即第一步的输入），中间步骤的结果图软删除
- **失败处理：** 若第二步失败，第一步的结果保留为正常生成图，不执行回滚
- **取消行为：** 取消链中任一步骤时，后续步骤级联取消
- **重试行为：** 重试 `chain_order=0` 的步骤会重建整条链；重试 `chain_order>0` 的步骤时，若存在失败/取消的前置步骤则重建整条链，否则创建独立任务
- **卡片显示：** TaskCard 显示 Link2 链式徽标 + 链信息行（格式：`1. 图生图 → 2. 高清放大`，仅步骤号 + 类别标签，不含工作流名称和状态），链内子任务在队列中视觉缩进

**添加任务时：** 弹出选择"立即执行"或"加入队列"

**导航角标（优先级从高到低，仅显示最高优先级的一个）：**
- 运行中：旋转动画（蓝色 Loader）
- 待开始：蓝色数字角标（pending 数量）
- 失败：红色数字角标，归零需手动清除失败任务
- 完成：绿色数字角标，进入任务队列页后清零

- 提交时调用 `PUT /api/workflows/:id` 更新（而非 `POST` 创建）

**导入对话框三步流程**：
1. **上传**：拖拽/选择 ComfyUI API JSON 文件 → 解析 `@` 前缀节点
2. **配置**：填写名称/类别/描述，将 Category 契约参数映射到解析出的节点参数（必填参数未映射时禁用提交）。选择类别后自动匹配：节点的 `@` 标签名**包含**契约参数名即视为匹配（如 `@my_prompt` 匹配参数 `prompt`），无需完全相等。配置区下方为「自定义参数分配」区域（见下文）
3. **提交**：注册工作流，重名时提示是否覆盖

**导入进行中保护**：导入过程中，点击对话框外部或按 Escape 键不会关闭对话框，防止意外中断导入流程

**`@` 前缀约定**：
- 工作流 JSON 中 `_meta.title` 以 `@` 开头的节点才会被解析
- `@` 标记的 LoadImage → 图片输入
- `@` 标记的 SaveImage/PreviewImage/ImageAndMaskPreview 等图像输出类节点 → 输出节点（图像输出节点同时加入 `text_outputs` 供导入 UI 分配）
- 其他 `@` 节点 → 进入「自定义参数分配」列表

**选择性输出捕获**：只有在导入时被显式 `@` 标记并分配了角色的输出节点才会被捕获。图像输出存入 `result_outputs` 格式 `{"type": "image", "path": "..."}`

**自定义参数分配（统一 UI）**：

对每个未被契约映射消耗的 `@` 标记节点，用户可为其分配角色：

| 节点类型 | 可用角色 | 说明 |
|----------|----------|------|
| LoadImage（未映射） | 不使用 / 输入 | 设为「输入」后展开图片输入配置：类型切换（图片 / 遮罩）+ 标签编辑。图片类型运行时显示 FaceRefPicker，遮罩类型显示 MaskEditor |
| 其他节点 | 不使用 / 输入 / 输出 / 输入+输出 | 「输入」展开标量参数列表（可勾选/编辑标签），「输出」展开输出标签编辑框 |

- 若 Category 定义了 `outputs`（如 preprocess 的 caption），对应节点自动设为「输出」角色；其余默认「不使用」
- 解析结果（解析摘要）区域默认折叠，点击可展开查看

**Manifest 格式**：
```json
{
  "mappings": { "param_name": { "node_id": "1", "key": "image", "type": "image" } },
  "output_mappings": { "反推提示词": { "node_id": "15", "key": "text" } },
  "extra_params": [
    { "name": "prompt.text", "label": "text", "type": "string", "node_id": "10", "key": "text" },
    { "name": "ref_image", "label": "参考图", "type": "image", "node_id": "5", "key": "image" },
    { "name": "mask_image", "label": "遮罩", "type": "image", "node_id": "6", "key": "image", "source": "file_path" },
    { "name": "model.model", "label": "模型", "type": "string", "node_id": "27", "key": "model", "choices": ["model_a.safetensors", "model_b.safetensors"] }
  ]
}
```
- `output_mappings` 中外层 key 为显示标签（任务详情中展示），内层 `key` 固定为 ComfyUI 节点的输出字段名（通常为 `text`）
- `extra_params` 中标量参数的 `name` 格式为 `节点名.参数键`，`label` 为用户可编辑的显示名称，`type` 为参数类型（string/int/float/bool/image）
- `extra_params` 中 `choices` 字段（可选）：当 ComfyUI 节点的输入为 combo/dropdown 类型时，导入阶段自动从 `/object_info` 获取可选值列表并存入 manifest。运行时渲染为 `<Select>` 下拉框而非文本输入
- 导入时自动查询 ComfyUI `/object_info` API 获取 @-tagged 节点的 combo 选项，结果缓存在内存中（模块级），ComfyUI 离线时回退到缓存
- `extra_params` 中 `type: "image"` 的条目来自未映射的 LoadImage 节点：无 `source` 字段时运行时使用 FaceRefPicker 选择媒体 ID，`source: "file_path"` 时使用 MaskEditor 绘制遮罩
- image 类型的 `extra_params` 不从 workflow_json 提取默认值（ComfyUI 占位文件名无意义）

## 6. 导入流程 `[P0]`

### 6.1 导入触发方式

- **粘贴导入：** 捕获全局剪贴板粘贴事件（`Ctrl+V`），导入单张图片，保存至 `AppData/imports/clipboard/`
- **文件选择（桌面端）：** 唤起系统文件选择器（tkinter），支持选择单张/多张媒体文件或文件夹。文件选择器默认显示"媒体文件"过滤器，包含图片和视频格式
- **文件上传（移动端）：** 移动端（触屏设备）使用 HTML `<input type="file" multiple accept="image/*,video/*">` 替代 tkinter 文件选择器，文件通过 HTTP multipart 上传到服务器，保存至 `AppData/imports/upload/`。移动端不支持子文件夹模式

### 6.2 导入逻辑

**选择单张或多张图片：**
1. 询问关联人物（链接到已有人物 / 新建人物 / 暂时无人物）
2. 询问是否新建图集或暂时无图集

**选择无子文件夹的文件夹：**
1. 将该文件夹视为一个图集导入
2. 询问是否更改图集名称（默认使用文件夹名）
3. 询问关联人物

**选择有子文件夹的文件夹（递归处理）：**
1. 展开导入确认弹窗，列出所有子文件夹（多层嵌套全部展平）
2. 每行显示：文件夹路径 → 图集名称（可编辑）→ 所属人物（可选）
3. 用户逐行确认或批量设置后提交

**文件夹名称智能拆分：**
- 当文件夹名包含下划线 `_`、横杠 `-` 或空格时，按**第一个**分隔符拆分
- 分隔符左侧作为默认**人物名**，右侧作为默认**图集名**
- 示例：`Alice_Wedding` → 人物"Alice"，图集"Wedding"；`Bob-旅行照片` → 人物"Bob"，图集"旅行照片"
- 若文件夹名不含上述分隔符，则人物名和图集名均默认为完整文件夹名（原有行为）
- 此规则适用于所有导入场景：无子文件夹的单文件夹、有子文件夹展平后的每一行、选择多张文件时的父文件夹名

**人物自动匹配与去重：**
- 拆分或推导出的人物名若与已有人物**名称匹配**（不区分大小写），自动选中该已有人物，而非新建
- 多个子文件夹拆分出相同人物名时（如 `Alice_生活照` 和 `Alice_写真集`），仅在第一次创建人物，后续自动复用，不会创建重复人物
- 用户仍可在导入弹窗中手动修改人物关联

### 6.3 导入进度与取消

- **进度显示：** 导入过程中显示进度条和计数（如"正在导入 45 / 230"）
- **可取消：** 导入过程中可随时取消（点击"取消导入"按钮），已导入的记录保留（不回滚）
- **导入中保护：** 导入进行中时，点击对话框外部或按 Escape 键不会关闭对话框，防止意外中断
- **大批量处理：** 超过 500 张图片时转为后台任务，顶部显示进度通知，用户可继续操作其他功能

### 6.4 人物上下文默认值

- 在**主页**触发导入：询问用户选择人物
- 在**人物主页**触发导入：默认关联当前人物，用户可强制更改

### 6.5 去重

- 以文件绝对路径作为唯一标识去重
- 重复扫描时跳过已存在记录，不重复导入
- **导入前预览：** 选择文件夹/文件后，导入界面立即显示媒体总数和已存在数量（异步扫描，不阻塞 UI）
  - 单文件夹/选择文件模式：来源区域下方显示"共 X 个媒体文件，其中 Y 个已存在将跳过"
  - 子文件夹模式：每个子文件夹行显示该文件夹的媒体数和已存在数；标题汇总全部文件夹的总数和已存在数
  - 全部已存在的文件夹自动排到列表末尾并降低不透明度，优先展示含新文件的文件夹
  - **子文件夹自动取消勾选**：扫描完成后，若子文件夹内所有文件均已存在（`existing >= total`），该子文件夹的复选框自动取消勾选
- **逐个配置模式已导入标记**：逐个配置（per-media）和子文件夹逐个视图中，后端 `POST /api/media/list-files` 返回每个文件的 `existing` 状态；已导入的文件在文件名旁显示"已导入"琥珀色标签，复选框置灰禁用且默认不勾选；全选操作自动跳过已导入文件
- **空图集清理**：导入流程中新创建的图集 ID 被追踪，导入完成后（包括错误/取消情况）自动检查并删除未导入任何媒体的空图集，避免产生垃圾数据
- **回收站重导入**：导入已在回收站中的文件时，自动恢复软删除记录（而非创建重复记录），更新 person_id 和 album_id 为新目标

### 6.6 文件丢失处理 `[P1]`

- 系统检测到文件路径不可访问时，卡片显示丢失标记（红色虚线边框 + 警告图标）
- 大图浏览（LightBox）中图片加载失败时，显示错误占位 UI（AlertTriangle 图标 + "文件不存在或无法加载"提示文字）
- 支持多入口重新定位：
  - **MediaCard 右键菜单**：缺失文件的卡片右键菜单显示"重新定位文件"，唤起系统文件选择器选择新路径
  - **MediaDetailDialog**：详情弹窗检测到文件缺失时，顶部显示警告横幅 + "重新定位"按钮
  - **LightBox 右键菜单**：大图浏览中右键"重新定位文件"（仅本地来源文件显示）
  - **图集详情页行模式右键菜单**：缺失文件可右键"重新定位文件"
- **批量重定向：** 设置页或图集右键菜单，批量更新某图集/文件夹的路径前缀

### 6.7 导入弹窗设计原则

- 所有弹窗复用同一套设计组件（标题栏、操作区、确认/取消按钮）
- 弹窗层级最小化，避免嵌套弹窗
- 保持轻量，减少性能占用

## 7. AI 功能详细需求

### 7.1 通用规范

- 所有 AI 功能通过 **WorkflowRunDialog**（居中弹窗）配置，替代旧的底部抽屉面板
- **统一流程**：右键菜单 → AI 功能 → 选择类别（upscale / face_swap / inpaint / image_to_image / text_to_image / preprocess） → WorkflowRunDialog 弹出 → 自动按 category 筛选已注册的工作流 → 动态渲染参数表单 → 提交任务
- WorkflowRunDialog 布局：标题（category label）→ 源图预览 → 工作流选择器（默认选 `is_default` 的工作流）→ 参数表单 → 底部"加入队列"/"立即执行"按钮
- 参数表单根据 category 契约参数 + manifest extra_params 动态渲染：
  - `image` + name 为 `source_image`/`base_image` → 自动填入右键所选图片（只读）
  - `image` + `source: "file_path"`（如 `mask`）→ 遮罩参数，显示"绘制遮罩"按钮，点击打开 MaskEditor，完成后回填 mask_path
  - `image`（其他，如 `face_ref`）→ FaceRefPicker 选取
  - `string` + `choices` → `<Select>` 下拉框、`string`（无 choices）→ textarea、`int`/`float` → number input、`bool` → toggle 滑块开关
- **共享参数表单组件（WorkflowParamForm）**：WorkflowRunDialog 和 AI 工具 Tab 共用同一参数渲染组件，确保所有入口的参数表单行为一致
- 任务提交后弹窗关闭，结果生成后自动存入 `AppData/generated/` 对应子目录
- 生成结果作为生成图关联至来源图（人脸参考图为准）

### 7.2 高清放大 `[P1]`

**工作流：** 分块高清放大工作流（分块逻辑对用户透明，由 ComfyUI 节点自动处理）

**自动反推提示词：** 工作流内置 `llama_cpp_instruct_adv` + Qwen3-VL 反推节点，自动从输入图像生成正向提示词（Extreme Detailed 风格），无需用户手动输入

**触发：** 右键 → AI 功能 → 高清放大 → WorkflowRunDialog 弹出（category=`upscale`），自动选中默认工作流，source_image 自动填入。参数由工作流 manifest 决定（如放大倍数、降噪强度等）

**结果处理：** 存入 `generated/upscale/`，链接至原图

### 7.3 局部修复 `[P1]`

**工作流：** `inpaint` 类别下可注册多个工作流（如 Flux、SDXL、Klein 等），用户在 WorkflowRunDialog 中选择

**流程：** 右键 → AI 功能 → 局部修复 → WorkflowRunDialog 弹出（category=`inpaint`） → 选择工作流 → 点击"绘制遮罩"按钮 → MaskEditor 全屏打开 → 绘制蒙版 → 确认 → 返回 Dialog → 配置其他参数（如提示词、降噪等，由工作流 manifest 决定） → 提交

**MaskEditor**：纯遮罩绘制工具，不包含任务提交逻辑。完成后返回 mask blob 给 WorkflowRunDialog，由 Dialog 负责上传遮罩和提交任务

**典型参数（由各工作流 manifest 定义）：**
- Flux 工作流：提示词（中文）、绘制强度、启用后位LoRA
- SDXL 工作流：绘制强度（无需提示词）
- Klein 工作流：无参数（无提示词、无强度）

**结果处理：** 存入 `generated/inpaint/`，链接至原图

### 7.4 换脸 `[P1]`

**工作流：** `face_swap` 类别下注册的工作流（如 qwen edit 换脸工作流）

#### 单张换脸

**流程：** 右键 → AI 功能 → 换脸 → WorkflowRunDialog 弹出（category=`face_swap`） → base_image 自动填入 → 通过 FaceRefPicker 选择 face_ref → 配置其他参数 → 提交

**结果处理：** 存入 `generated/face_swap/`，生成链 `parent_media_id` 指向底图（结果是底图的衍生）；**归属默认继承人脸参考图的人物/图集**（结果展示的是参考人脸）。WorkflowRunDialog 提供「结果归属」下拉选项，可切换为底图所属人物。

**交换底图与人脸参考：** WorkflowRunDialog 中两个图片参数之间提供快捷交换按钮，一键互换底图与人脸参考。

#### 批量 AI（通用）

**触发入口：**
- 图集右键菜单 → AI 功能 ▸ 批量 AI → 选择工作流类别
- 人物主页图集卡片右键菜单 → 同上
- 多选工具栏 → AI 功能按钮 → 选择工作流类别

**可批量执行的工作流类别：** `upscale`（高清放大）、`face_swap`（换脸）、`image_to_image`（图生图）、`preprocess`（预处理）
**不可批量执行：** `inpaint`（需要逐张绘制遮罩）、`text_to_image`（无源图）

**流程（BatchAiDialog）：**
1. 弹出 BatchAiDialog，展示将要处理的图片数量（仅本地图/截图，跳过 AI 生成图）
2. 选择 AI 类别（仅可批量类别）
3. 选择具体工作流（从该类别下已注册的工作流中选取）
4. 配置共享参数（WorkflowParamForm，隐藏源图和遮罩参数，所有图片共用的参数如人脸参考图等）
5. 提交批量任务

**后端 API：** `POST /api/tasks/batch`，按图片数量创建独立任务，共享参数统一、每张图片替换 `source_param_name` 对应的参数值。返回 `{tasks_created}`。

**结果处理：** 每张生成图加入对应原图的生成链（`parent_media_id`），放回原图所在图集，不额外创建结果图集

### 批量+链式

批量处理支持三种模式：
1. **独立任务**：每张图创建独立任务（现有行为）
2. **复合工作流**：选择复合工作流时，每张图展开为链式任务
3. **手动链式**：通过 `chain_step` 参数为每张图附加后续步骤

API 返回新增字段：
- `chains_created`: 创建的链式任务组数量
- `skipped_generated`: 跳过的 AI 生成图数量（media_ids 模式）

### 7.5 裁剪与剪辑 `[P0]`

#### 图片裁剪（CropEditor）

**触发：** 图片卡片右键菜单 / LightBox 右键菜单 → "裁剪"

**CropEditor 弹窗：**
- 预览画布 + 裁剪框（可拖拽、调整大小）
- 宽高比预设按钮：自由、1:1、4:3、3:4、16:9、9:16、原始比例
- 旋转：90° 顺时针/逆时针旋转
- 底部操作：「覆盖原图」复选框（默认关闭）+ 取消 + 确认裁剪
- 确认后调用 `POST /api/media/{mid}/crop`，overwrite=true 时覆盖原文件，否则存入 `generated/crop/` 作为新媒体（parent_media_id 指向原图）

#### 工作流临时裁剪

**触发：** WorkflowRunDialog 中源图预览区的裁剪按钮

**流程：**
1. 点击裁剪按钮 → 弹出 CropEditor（无覆盖选项）
2. 确认裁剪 → 调用 `POST /api/media/{mid}/upload-crop` 上传裁剪结果到 `cache/crops/`
3. 返回 WorkflowRunDialog，source_image 替换为裁剪后的临时路径
4. 临时裁剪文件由后端定期清理

#### 视频剪辑（VideoTrimEditor）

**触发：** 视频卡片右键菜单 / LightBox 右键菜单 → "剪辑"

**VideoTrimEditor 弹窗：**
- 视频预览 + 时间轴拖拽选择起止点
- 起始/结束时间输入框（可手动输入精确时间）
- 精确模式开关：关闭时使用 ffmpeg stream copy（快速，关键帧对齐）；开启时 re-encode（精确到帧，较慢）
- 确认后调用 `POST /api/media/{mid}/trim`，结果存入 `generated/trim/` 作为新媒体（parent_media_id 指向原视频）

1. 触发后弹出底部抽屉参数面板
2. 参考图默认为当前选中图，可替换

## 8. 图片管理功能

### 8.1 评分系统

- 评分范围：1-5 星，可清除（清除后视为未评分）
- **平均分计算规则：所有未删除图片参与计算，已评分媒体使用实际评分，未评分媒体按 2.5 分计入加权平均**
- 全部媒体数为 0 时，avg_rating 为 null，UI 显示"未评分"
- 前端显示格式：`★4.2 (12)` — 平均分 + 已评分数量
- 图集和人物的平均评分实时计算（基于下属未删除图片的加权平均）
- 评分入口：大图模式顶部、图片右键菜单、多选批量评分

### 8.2 筛选系统

筛选条件可叠加，在所有层级生效（包括大图模式的上下张切换；但来源类型筛选不影响大图切换逻辑，以保证生成链遍历完整）：

| 维度 | 选项 | 适用页面 |
|------|------|----------|
| 评分 | 全部 / 5星 / 4星+ / 3星+ / 2星以下 | 人物库、图集列表、未分类、图集详情 |
| 来源类型 | 全部 / 本地图 / AI生成 / 截图 | 未分类、图集详情 |
| 标签 | 全部 / 各已有标签（多选） | 人物库、人物主页图集列表 |

**筛选默认值配置：** 在设置页"筛选默认值"区域可配置评分筛选和来源类型的默认值（默认均为"全部"），存储在 localStorage `motif-filter-defaults`。每次进入页面时筛选条件自动重置为设置页配置的默认值。

### 8.3 排序系统

| 层级 | 排序选项 | 默认值 |
|------|----------|--------|
| 人物库 | 最新创建 / 最早创建 / 评分最高 / 评分最低 / 名称 A-Z / 名称 Z-A | 最新创建 |
| 图集列表 | 最新创建 / 最早创建 / 评分最高 / 评分最低 / 名称 A-Z / 名称 Z-A | 最新创建 |
| 未分类 | 最新添加 / 最早添加 / 评分最高 / 评分最低 | 最新添加 |
| 图集详情 | 默认顺序 / 默认倒序 / 最新添加 / 最早添加 / 评分最高 / 评分最低 | 默认顺序 |

**双向排序：** 所有排序选项均支持升序和降序两个方向。前端排序值格式为 `field:dir`（如 `created_at:desc`），后端 API 通过 `sort_dir` 参数（`asc`/`desc`）控制方向。

**评分排序规则：** 降序时有评分的按评分降序排列，未评分（avg_rating = null）的排末尾；升序时未评分排最前，有评分的按评分升序排列。未评分组内按导入时间降序

**排序默认值配置：** 在设置页"筛选默认值"区域可为每个页面单独配置默认排序方式，存储在 localStorage `motif-filter-defaults`。每次进入页面时排序自动重置为设置页配置的默认值。

### 8.4 标签系统 `[P0]`

人物和图集均支持多标签分类，便于跨人物/跨图集的分组浏览和筛选。

**标签属性：** 名称（唯一）+ 颜色（可选，默认灰色）+ 排序权重

**标签编辑器（TagEditor）：**
- 触发方式：PersonCard / AlbumCard 右键菜单"管理标签"
- 弹出 Popover，列出所有标签（带颜色圆点），勾选/取消关联
- 顶部搜索框可过滤标签，输入不存在的名称时显示"创建标签"选项
- 新建标签时自动分配默认颜色，可在设置页"标签"Tab 修改

**筛选栏标签 chips：**
- 人物库 FilterBar 显示标签筛选区，以彩色 chip 形式展示所有标签，支持多选
- 选中标签后仅显示包含任一选中标签的人物（OR 逻辑）
- 人物主页图集区 FilterBar 同理，按图集标签筛选

**PersonHome Hero 标签展示：** 人物姓名下方显示该人物的标签 chips，点击可跳转到人物库并按该标签筛选

**标签管理：** 设置页"标签"Tab 提供全局标签管理（新建、重命名、改色、排序、合并、删除），详见 §5.11.3

### 8.5 封面管理

- 人物封面：默认第一张图，右键图片"设为人物封面"
- 图集封面：默认第一张图，右键图片"设为图集封面"
- 视频封面：默认第 1 帧；有截图后使用第一张截图；多张截图时可手动指定

### 8.6 生成链管理 `[P1-后续]`

- 生成图支持多级嵌套（生成图的生成图）
- 任意生成图可脱离链接变为本地图（子数据：评分、二次生成图全部跟随）
- 脱离时物理文件不移动，数据库中解除关联关系，source_type 改为 `local`
- **脱离 API**：`POST /api/media/{id}/detach`，将 `parent_media_id`、`workflow_type`、`generation_params` 置为 null，`source_type` 改为 `local`。子代（children、grandchildren）保持与被脱离图的关联，形成新的独立生成树

### 8.7 软删除与回收站

- 删除操作均为软删除（标记 `is_deleted = true`，记录 `deleted_at`）
- 本地图：任何情况下只删除记录，物理文件不动
- 生成图/截图：软删除后进入回收站；回收站永久删除时同时删除物理文件
- **生成链删除模式选择：** 删除有子图的媒体时，弹出 DeleteChoiceDialog 让用户选择：
  - **级联删除（cascade）**：一并删除该图片的所有子图（默认）
  - **保留子图（reparent）**：仅删除该图片，其子图的 `parent_media_id` 指向被删图的父节点（即子图"上移"到祖父节点下）
  - 批量删除时，若选中项包含生成链图片，同样弹出选择对话框
  - 无子图的图片直接走简单确认对话框（ConfirmDialog）
- 回收站支持恢复到原位置
- 回收站手动清空
- **自动清理：默认 30 天后自动永久删除，保留天数可在设置页配置（设为 0 关闭自动清理）**

### 8.8 批量清理低分生成图 `[P1-后续]`

**功能：** 批量删除当前作用域内评分低于指定值的生成图

**作用域：**
- 主页（全局）：清理所有人物下的低分生成图
- 人物主页：清理该人物下的低分生成图
- 图集详情页：清理该图集下的低分生成图

**触发入口：** 各层级右键空白处菜单

**清理规则：**
- 仅清理生成图（source_type 为 `generated` 或 `screenshot`），本地图不受影响
- **未评分的生成图永远不参与清理**
- 评分严格小于指定阈值的生成图将被删除

**操作流程：**
1. 弹出阈值设置面板（选择评分阈值，如"清理低于 3 星的生成图"）
2. 展示预览列表：将被删除的生成图缩略图及总数量
3. 用户二次确认后执行软删除（进回收站）

### 8.9 随机探索

**功能：** 在指定作用域和筛选条件内随机排列媒体，以大图模式浏览

**触发入口：** 各层级顶部工具栏"随机探索"按钮，自动继承当前作用域和已设筛选条件

**作用域：**
- 从触发层级自动继承（主页 = 全局，人物主页 = 该人物，图集页 = 该图集）
- 进入后可手动切换作用域（全局 / 指定人物 / 指定图集）

**筛选选项（可叠加）：**
- 评分：未评分 / 大于等于指定星级
- 媒体类型：图片 / 视频
- 来源类型：本地图 / 生成图

**探索行为：**
- 进入时对符合条件的媒体随机洗牌，生成本次探索序列
- 筛选条件变更后需手动点击"重新洗牌"（reshuffle），不动态应用
- 水平轴（← →）在 shuffled 序列中顺序切换（扁平导航），垂直轴（↑ ↓）仍可进入当前图的生成链
- 评分操作实时生效，但不重排本次序列

**界面：** 复用大图模式，顶部额外显示"探索进度"（如 12 / 87）和"重新洗牌"按钮

**沉浸模式：** 可在随机探索中开启沉浸模式（见 5.5.1 节）

## 10. 技术实现要点

### 10.1 工作流类型与命名规范

**workflow_type 枚举及文件名缩写：**

| 功能 | workflow_type | 文件名缩写 |
|------|--------------|-----------|
| 高清放大 | upscale | upscale |
| 局部修复 Flux（带提示词） | inpaint_flux | inpaint_f |
| 局部修复 SDXL（通用 A） | inpaint_sdxl | inpaint_s |
| 局部修复 Klein（通用 B） | inpaint_klein | inpaint_k |
| 换脸 | face_swap | faceswap |
| 图生图 | img2img | img2img |
| 写真生成 | portrait | portrait |
| DWPose 提取 | pose_extract | pose |

**文件命名规则（全局统一）：**
```
{参考图文件名}_{workflow_type缩写}_{序号}.png
示例：IMG_001_img2img_001.png
     IMG_001_faceswap_002.png
```

### 复合工作流

复合工作流将多个子工作流组合为一个独立条目。运行时自动展开为链式任务序列。

- 创建方式：工作流管理页 → 「创建复合工作流」按钮
- 步骤数量：2-5步，嵌套展开后总步数不超过 10
- 嵌套支持：复合工作流的步骤可以引用其他复合工作流，运行时递归展开为扁平链式任务序列
- 循环引用检测：DFS 检测，防止 A→B→A 死循环
- category：取首步工作流的 category（递归到叶子层）
- 列表显示：带「复合」标签和步骤数量
- 运行时：展开为 N 步链式任务，首步接收用户参数，后续步骤自动接收前一步输出

### 10.2 ComfyUI 集成

- 通过 `/prompt` 接口提交工作流 JSON
- 通过 WebSocket 实时监听任务进度（`/ws?clientId=...`），收到 `execution_success` / `execution_error` 后获取结果
- **WebSocket 健壮性：** 启用 heartbeat（30 秒 ping），空闲超时 120 秒（无消息则判定 ComfyUI 卡死，立即报错而非等到任务超时）；WebSocket 异常断开时抛出明确错误（"ComfyUI may have crashed or restarted"）
- 工作流 JSON 模板存储在 `AppData/workflows/` 目录下，每个工作流类型一个 JSON 文件
- 参数占位符格式：`{{param_name}}`，如 `{{input_image_path}}`、`{{denoise_strength}}`
- 后端在提交前将占位符替换为实际参数值
- 任务严格串行，FastAPI 维护队列，一次只向 ComfyUI 提交一个任务
- 单任务超时阈值：默认 10 分钟，可在设置页配置

### 10.3 任务队列

- SQLite 存储任务记录和队列顺序
- FastAPI 后台线程维护队列调度
- 支持 4 种启动模式：手动（manual）/ 自动（auto）/ Cron 定时（cron，需 `croniter` 库解析表达式）/ 延时（delay）
- 支持队列暂停（`is_paused`），暂停后不执行新任务，已运行任务不受影响
- 延时模式从最后一个任务加入队列时刻起算
- 任务失败自动跳过，继续执行下一个
- 失败任务保留错误信息，支持手动重试
- **启动恢复：** 服务重启时自动将遗留的 `running` 状态任务标记为 `failed`（错误信息"服务重启时任务仍在执行，已标记为失败"），防止僵尸任务阻塞队列
- 链式任务（chain）：同一 `chain_id` 的步骤由 queue_runner 原子执行，步骤间通过 `chain_source_param` 自动传递结果 media_id；全部成功后 reparent 最终结果到原始源图并软删除中间结果
- **自定义工作流输出兼容：** 当自定义工作流仅通过 `output_mappings`（PreviewImage 节点）输出图片、无 SaveImage 节点时，queue_runner 自动将输出图片提升为 Media 记录，确保链式传递和结果保存正常工作
- **独立任务校验：** 创建独立任务时校验 params 不含未解析的 `__chain_input__` 占位符，防止无效任务进入队列
- **链式默认工作流：** 点击「链接下一步」时默认选择 `upscale`（高清放大）类别及其默认工作流

### 10.4 文件系统监控

- 导入时记录文件绝对路径，不复制文件
- 访问文件时检测路径可用性，不可用时显示丢失标记
- 剪贴板导入图片保存至 `AppData/imports/clipboard/`

### 10.5 放大模式实现

- **架构**：提取为 `useImageZoom` hook（`frontend/src/hooks/useImageZoom.ts`），所有缩放状态存 ref，rAF 直接操作 img.style.transform，避免 React 重渲染
- **渲染**：`transform: translate(tx, ty) scale(s)` + `transform-origin: 0 0` + `will-change: transform`，动画缩放加 `transition: transform 0.3s ease-out`，手势时 `transition: none`
- **鼠标移动平移**：放大后鼠标移动即平移，公式：`nx = (mx - fitX) / fitW`（归一化坐标），`translateX = mx - nx * zoomW`，令鼠标在原图上的位置点与放大图对应位置重合；`nx/ny` 超出 `[0,1]` 时隐藏放大图（`opacity: 0`）并恢复默认指针
- **放大镜指针**：放大模式下鼠标在图片范围内时，CSS `cursor: url("data:image/svg+xml,...")` 将原生指针替换为 SVG 放大镜图标（hotspot 居中），浏览器原生渲染零延迟
- **无边界约束**：放大模式不使用 clamp，图片完全跟随鼠标位置映射
- **滚轮分区**：hook 在图片容器上 `e.stopPropagation()` 拦截滚轮做缩放，外层 document 监听器只收到黑边/缩略条的滚轮做切图
- **单击 vs 拖拽**：mousedown→mouseup 移动距离 < 5px 且时间 < 300ms = 单击
- **黑边点击关闭**：`isPointOnImage()` 判断点击位置，黑边点击不 `stopPropagation`，冒泡到外层 `closeLightbox()`
- **回调稳定性**：`onZoomChange` 存入 ref（`onZoomChangeRef`），避免外部传入不稳定回调导致 effect 链式重建
- **移动端**：同一 hook 处理双指捏合缩放+单指拖拽平移，与 TouchArbiter 协调
- 蒙版编辑器中禁用放大模式，改为画布整体缩放（滚轮缩放以指针为中心）

### 10.6 蒙版编辑器

- 使用 HTML5 Canvas 实现画笔绘制
- 蒙版以绿色半透明层叠加在图片上
- 支持撤销/重做（操作历史栈）
- 蒙版导出为 RGBA PNG（alpha=0 为修复区域，alpha=255 为保留区域），兼容 ComfyUI `LoadImage` 节点自动提取 alpha 通道作为 mask

### 10.7 视频处理

- HTML5 原生视频播放器，支持局域网串流
- 文件服务端点 `/api/files/serve` 必须支持 HTTP Range 请求（`206 Partial Content`），否则浏览器无法 seek 视频
- 截图使用 Canvas 从视频元素捕获当前帧
- 提供逐帧步进功能（前进/后退 1 帧）

### 10.8 生成链数据结构

- `parent_media_id` 自引用外键实现多级嵌套
- **最大递归深度 10 层**，后端在创建生成图时检查深度，达到上限时拒绝并提示"已达最大嵌套深度"
- 树状图渲染需递归查询子节点（后端提供 `/api/media/{id}/tree` 接口返回完整生成链）
- **生成链缓存：** LightBox 使用 `chainCache`（Map<rootId, {tree, flat}>）缓存已加载的生成链。缓存在以下时机失效：关闭 LightBox 时全量清除；删除媒体时清除；任务完成时（`completed_since_last_view` 递增）全量失效并自动重新加载当前链；脱离操作时全量清除并重载
- 脱离操作：更新 `source_type` 为 `local`、清空 `parent_media_id`、`workflow_type`、`generation_params`；被脱离图从 chainFlat 中移除（连同其后代），插入 localItems 成为新的本地图根节点，chainIndex 重置为 -1

**提取任务行为：**

### 11.2 网页图片抓取器 `[P3]`

**功能：** 解析社交平台帖子或含图网页链接，下载图片/视频并导入媒体库

**支持平台：**

| 平台 | 技术方案 | 授权方式 | 账号扫描 |
|------|----------|----------|----------|
| 小红书 | Playwright 无头浏览器（单条解析），Cookie HTTP（账号扫描） | 单条无需授权；账号扫描需 Cookie | 支持 |
| 抖音 | Cookie HTTP（RSC flight data 解析） | 账号 Cookie | 支持 |
| B站动态/专栏 | API 较开放 | 账号 Cookie（部分内容需要） | 未实现 |
| X / Twitter | API 限制严格 | 账号 Cookie 或 API Key | 未实现 |
| Telegram | 公开频道直接抓取，私有频道需授权 | Telethon 账号授权 | 未实现 |
| 通用网页 | requests + Playwright 渲染动态页 | 无需授权 | 不适用 |

**各平台授权配置：** 在设置页"平台 Cookie"分区配置。小红书单条链接解析无需 Cookie（Playwright 抓取），但账号扫描需要 Cookie；抖音所有功能均需 Cookie

#### 11.2.1 触发方式

**PC 端：**
- 在小工具页面的输入框粘贴链接，点击"解析"按钮

**手机端：**
- 同上，但粘贴内容可能包含分享提示文字（如"博主名发布了一条小红书笔记，快来看看吧~ http://xhslink.com/xxxxx"）
- 后端自动从粘贴文本中提取有效链接，支持多种平台的短链和完整链接格式

#### 11.2.2 下载流程

1. **粘贴文本 → 提取链接**
   - 后端从原始文本中正则提取有效 URL
   - 识别平台类型
   - 检查对应平台授权是否配置（小红书无需授权），未配置则提示前往设置

2. **解析元数据**（后台请求，前端显示加载状态）
   - 博主用户名 / 显示名
   - 帖子标题
   - 发布日期
   - 图片/视频预览（缩略图列表）
   - 媒体数量

3. **页面内展示解析结果，用户确认**

4. **账号关联步骤**
   - 检查该平台账号是否已有关联人物记录
   - **已关联**：自动带入关联人物，用户可更改
   - **未关联**：默认"新建人物"模式，人物名预填为博主显示名；可切换为"已有人物"或"暂不关联"
   - 提供"记住此账号的关联关系"选项（默认勾选，写入 PlatformAccount 表）

5. **图集配置**
   - 图集名称默认为帖子标题，可编辑
   - 选择归入方式：
     - 新建图集（默认）
     - 合并到已有图集（从该人物图集中选）
     - 作为未分类（不归入图集）
   - 同一账号再次下载时：默认新建图集，可选已有图集

6. **确认下载**
   - 图片存入 `AppData/downloads/{platform}/`
   - 视频同样下载
   - 走正常导入流程，写入 Media 记录和 DownloadRecord 记录
   - 下载完成后跳转至生成的图集

#### 11.2.3 下载记录页

- 展示历史下载记录列表（来源平台、账号、标题、日期、状态）
- 失败记录显示错误原因，支持重试
- 点击记录可跳转至对应图集

#### 11.2.4 账号批量扫描与下载

**功能：** 扫描某平台账号下的所有图文笔记，一键批量下载

**触发入口：**
- **小工具页**：解析单条链接后，显示"扫描该账号所有图文笔记"按钮
- **人物主页**：关联的平台账号显示为可点击的徽章，点击即触发扫描

**流程：**

1. **扫描阶段**（后台异步任务，前端 2 秒轮询）
   - 通过平台内部 API 分页获取所有图文笔记（跳过视频笔记），同时预收集每条笔记的完整图片 URL
   - 抖音：使用 `/aweme/v1/web/aweme/post/` API，按 `sec_user_id` 分页，筛选 `aweme_type=68`（图文笔记）
   - 小红书：使用 Playwright 加载用户主页，拦截浏览器 JS 自动签名的 `/api/sns/web/v1/user_posted` API 响应，滚动分页获取所有笔记；从响应中的 `image_list` 提取每张图片的 WB_DFT 高清 URL；需在设置页配置小红书 Cookie

2. **去重**
   - 扫描完成后，按 `note_id` 查询 `DownloadRecord` 表（从 `source_url` 提取），排除已成功下载且图集仍存在的笔记
   - 已删除图集的笔记（`album_id` 被 `SET NULL`）视为未下载，允许重新导入
   - `source_url` 统一存储为规范格式 `https://www.xiaohongshu.com/explore/{note_id}`，无论原始链接是 `/explore/` 还是 `/discovery/item/` 或带分享参数
   - 前端显示"X 个新笔记（已跳过 Y 个已下载）"
   - 全部已下载时提示"所有笔记都已下载过，没有新内容"

3. **确认下载**
   - **小工具页**：用户可配置人物关联（不关联 / 已有 / 新建）、图集模式（每笔记一图集 / 未分类）、记住账号
   - **人物主页**：默认关联当前人物、每笔记一图集模式，一键确认
   - 进度卡片仅在当前人物关联的平台账号匹配时显示（不会泄漏到其他人物主页）

4. **批量下载**（后台异步，前端显示进度条）
   - 优先使用扫描阶段预收集的图片 URL 直接下载（避免逐条重新解析笔记页面，规避反爬限制）；仅在无预收集 URL 时回退到单条解析
   - 下载到 `AppData/downloads/{platform}/` → 仅在有文件下载成功时创建 Album → 创建 Media/DownloadRecord 记录
   - 每条笔记间延迟 1 秒防限流
   - 支持取消
   - 完成后自动刷新页面数据

**技术实现：**
- 后端 `batch_downloader.py`：内存中 `BatchJob` 状态机（scanning → scan_complete → downloading → completed/failed/cancelled）
- 进度通过 `GET /download/scan-jobs/{job_id}` 轮询，响应包含 `platform` 和 `username` 字段用于前端作用域限定
- 每个 Job 独立 asyncio Task，支持取消（`task.cancel()`）
- 小红书反爬策略：XHS 阻止 headless 浏览器直接访问笔记页面（即使带 Cookie 也 302 到 404），因此批量下载不逐条解析，而是在扫描阶段通过 API 拦截一次性收集所有图片 URL

## 13. 异常处理规范

### 13.1 ComfyUI 异常

| 场景 | 预期行为 |
|------|----------|
| ComfyUI 进程未启动 | 前端显示"ComfyUI 启动中..."，AI 功能按钮禁用（"加入队列"仍可用），每 5 秒轮询连接状态 |
| ComfyUI 进程崩溃 | 后端检测到连接断开后标记状态为"连接失败"，当前运行中的任务标记为 `failed`（error_message = "ComfyUI 连接中断"），后端每 10 秒尝试重连，连续失败 30 次后停止重试并提示用户手动检查 |
| ComfyUI 任务超时 | 单任务超时阈值 10 分钟（可配置），超时后标记 `failed`，跳至下一任务 |

### 13.2 存储异常

| 场景 | 预期行为 |
|------|----------|
| 磁盘空间不足 | 生成任务执行前检查剩余空间（阈值 500MB），空间不足时拒绝执行并提示"磁盘空间不足，请清理后重试" |
| 文件路径不可访问 | 卡片显示丢失标记（红色虚线边框 + 警告图标），不阻塞其他功能 |
| 缩略图生成失败 | 显示默认占位图，不影响其他操作，后台重试一次 |

### 13.3 网络异常（局域网场景）

| 场景 | 预期行为 |
|------|----------|
| 客户端与后端断连 | 前端显示"连接中断，正在重连..."横幅，每 3 秒重试，恢复后自动刷新当前页面数据 |
| 请求超时 | 前端操作 10 秒无响应显示"操作超时"提示，用户可点击重试 |

### 13.4 外部平台异常（网页抓取器） `[P3]`

| 场景 | 预期行为 |
|------|----------|
| Cookie 过期 | 提示"授权已过期，请在设置页更新 Cookie" |
| 平台限流/封禁 | 显示具体错误信息，建议等待后重试，记录失败到 DownloadRecord |
| 内容已删除/不可访问 | 提示"原始内容不可访问"，记录失败原因 |
